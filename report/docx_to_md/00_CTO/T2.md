# 🎯 [Task] 核心架构修复：Markdown 段落边界重建与文献排版规范化 (v1.2)

## 📌 背景说明 (Background)
在 Phase 1 阶段的大规模 DOCX 转 MD 测试中，我们发现当前的 XML 解析管线存在严重的**语义物理边界丢失**问题。长篇学术手稿被压缩成了没有换行的“文字墙”，且 Zotero 自动生成的参考文献发生行内粘连。
这不仅破坏了视觉阅读体验，更会**直接摧毁下游无视觉 AI Agent 的注意力机制**，导致 RAG 检索分块时发生灾难性的语义撕裂。本任务旨在通过底层修复与正则拦截，彻底规范输出格式。

---

## 🛠️ 需求拆解 (Requirements)

### 需求一：重建段落语义边界 (Paragraph Boundary Restoration)
* **缺陷描述**：当前解析器在遍历 `<w:p>`（段落）节点时，仅对空段落执行了 `flush`，导致连续的有字段落被错误拼接。
* **开发要求**：
  1. 重构文档遍历逻辑，确保在处理完每一个有效的 `<w:p>` 闭合节点后，强制追加 Markdown 标准的段落分割符（`\n\n`）。
  2. 确保在追加换行符前进行 `.strip()` 判空，避免因文档中冗余的空段落产生过多的无意义换行。

### 需求二：参考文献的“双层火控”换行隔离 (Reference Normalization)
* **缺陷描述**：Zotero 等文献管理插件在自动生成文献时，常常使用软回车（Shift+Enter, 即 `<w:br/>`）或无换行混排格式，导致文献 `[1]`、`[2]` 粘连在同一行。
* **高优风控提示**：绝对**禁止使用全局正则**进行强行切分，否则会误杀正文中的正常学术引用（如：“在 Agri-CPS 系统中 [12]”）。
* **开发要求（实施作用域隔离策略）**：
  1. **第一层（遍历时拦截）**：在深入解析段落子节点（如 `<w:r>`）时，必须显式捕获 `<w:br/>` 标签，并将其硬转译为 `\n`。
  2. **第二层（安全正则清洗）**：在输出 Markdown 字符串前，动态寻找文档尾部的参考文献区域锚点。
     * ⚠️ **双语与多级标题兼容**：锚点定位必须同时兼容中文 (`参考文献`) 和英文 (`References`)，并考虑一级 (`# `) 和二级 (`## `) 标题的情况。
     * **精准清洗**：切割文档后，**仅对该锚点之后的区域**执行正则清洗（前向断言扫描 `[1]`, `[2]` 等引文标识，前方非换行则强制插入 `\n`），再拼接回正文。确保正文内容 0 污染。

---

## ✅ 验收标准 (Definition of Done)
1. **零文字墙**：长篇论文的自然段落必须与 Markdown 中的段落（双换行）完美对应。
2. **文献独立**：References / 参考文献 列表下的每一条引文必须独立占据新的一行，格式规整。
3. **无次生破坏**：原有 v1.0 / v1.1 版本的公式提取管线不受影响；**正文中的行内引用 `[N]` 绝对不能被错误换行打断**。
