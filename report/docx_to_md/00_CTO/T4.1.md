# 🚀 研发指令：Task 4.1 - 协议本源回归与全局字节流重构 (The First Principles)

**致核心研发团队：**

经过极其深度的底层字节流排查，我们终于迎来了本次 MTEF 解析器开发的“阿波罗时刻”。通过抓取真实的 Hex Dump（如 `01 00 02 04 84 B8`），我们确诊了导致整个 AST 空间拓扑坍缩、以及产生大面积 CJK 生僻字的根本元凶：**我们在最底层对 MTEF 协议的物理认知出现了全局性偏差。**

官方协议**根本没有使用位域压缩（Bitfield Packing）**。
我们之前使用的 `options = tag >> 4` 是一次致命的“刻舟求剑”，这导致解析器在读取每一个标准节点时，都**少读了一个字节**。这一步之差，摧毁了状态机在二进制数据流中行走的因果链，导致后续的 Template 节点参数被错位解析，最终引发了多米诺骨牌式的全盘崩溃。

现在，我正式下达 **Task 4.1**，要求大家立刻对 `mtef_parser.py` 进行底层的“洗髓伐骨”。这不是修补，而是做减法。

### 🛠️ 架构重构指令 (Refactoring Directives)

请彻底抛弃补丁思维，严格按照“确定性递归下降状态机”的绝对法则，落实以下三项重构：

#### 1. 废除掩码，确立“双字节包头” (Two-Byte Header Strictness)
在主循环 `_parse_line` 以及任何涉及标准 AST 节点读取的地方，彻底删掉所有的位运算拆分逻辑（`tag & 0x0F` 和 `tag >> 4`）。
* **强制规范**：所有标准节点（LINE, CHAR, TMPL, PILE, SIZE 等），其头部必须实施两次独立的单字节读取：
  1. 第一步：`tag = self._read_byte()` （确定节点类型）
  2. 第二步：`options = self._read_byte()` （确定节点的独立配置项）
* **游标纪律**：只有吃完这两个字节，才能进入后续的 Payload（数据段）判断。

#### 2. 大清洗：拔除所有的“防御性补丁” (Purge the Band-Aids)
我们之前写的很多代码，其实都是在为游标少走的那一步“擦屁股”。现在步长对了，这些补丁只会成为新的绊脚石。
* **删除重对齐逻辑**：在 `_read_char_value` 中，立刻删掉那个关于 `typeface=0` 时尝试窥探后一个字节并重新对齐的防御代码。回归最纯粹的读取逻辑。
* **重估预处理器**：检视之前加上的 `_consume_size_prefix`。既然不会再有因为错位导致的假 Tag 乱入，这个拦截器如果还在发挥作用，它拦截的必须是真正合法的独立 SIZE 记录，绝不能用来“吞噬”错位产生的垃圾字节。

#### 3. 闭眼消费与信仰之跃 (The Consumption Contract)
修改完双字节包头后，必须仔细复核每个 Tag 分支下的 Payload 消费逻辑。
* 当读完 `Tag` 和 `Option` 后，手册规定要读几个字节（比如 `CHAR` 在特定 Option 下读 1 个或 2 个字节，外加 1 个 Typeface 字节），就**不假思索地往前走几步**。
* 严禁在消费数据时向后张望或猜测下一个 Tag 是什么。只要你们的步长计算严丝合缝，下一步踩下去的地方，在物理上 100% 就是下一个合法的帧头。

### 🎯 验收指标 (Acceptance Criteria)

这是一场不破不立的战役。我不要看到任何妥协。

1. **生僻字彻底绝迹**：去掉重对齐补丁后，重新运行测试。`뢄` 和 `햄` 必须被正确路由回我们做好的 MTCode 映射表，输出纯净的 $\theta_p$ 和 $\theta_y$。
2. **拓扑维度自然复苏**：游标纠正后，`TMPL` 模板会在正确的时间点被正确触发。`\langle\rangle\Theta` 这种倒置结构必须消失，上下标结构（如 $x_{left}$）必须在 `_template_to_latex` 中自然生成，无需任何特殊干预。

拔掉那些阻碍游标前进的杂草，让状态机在干净的二进制通道里跑起来！
