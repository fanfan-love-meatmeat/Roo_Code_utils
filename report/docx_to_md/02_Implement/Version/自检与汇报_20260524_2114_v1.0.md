## 2. 热修复：三项缺陷定点修复

### 🚨 修复 1: python-docx Paragraph 实例化时 parent 上下文丢失

| 项目 | 内容 |
|---|---|
| **原缺陷** | `Paragraph(child, body_element)` — `body_element` 是 lxml `_Element`，无 `part` 属性，访问 `.style` 时触发 `AttributeError` |
| **修复** | 函数签名改为 `_parse_body_elements(doc, ...)`，注入 `doc._body` 作为父级包装器 |
| **影响行** | L503-511 (`_parse_body_elements` 签名 + `body_wrapper`)，L518-545 (Paragraph/DocxTable 实例化)，L598 (调用点) |

### 🚨 修复 2: 表格内 oMath 公式换行符泄漏

| 项目 | 内容 |
|---|---|
| **原缺陷** | `oMathPara` 在表格内做了换行展平，但 `oMath`（行内公式）没有 — 含矩阵的行内公式的 `\n` 会被 `_parse_table` 的 `cell_text.replace("\n", "<br>")` 错误替换，导致 LaTeX 矩阵语法损坏 |
| **修复** | `is_in_table=True` 时，`oMath` 分支同样执行 `latex.replace("\n", " ")` 展平 |
| **影响行** | L416-417 |

### 🚨 修复 3: XSLT 转换树的二次序列化

| 项目 | 内容 |
|---|---|
| **原缺陷** | Step1 产出 `mathml_tree` → `str()` → `etree.fromstring()` → Step2。lxml XSLT 引擎原生支持直接传递 `_XSLTResultTree`，序列化/反序列化冗余 |
| **修复** | 直接传递 `mathml_tree` 给第二步转换器；空产检查改为 `root.getroot()` + `len()` |
| **影响行** | L103-112 |
