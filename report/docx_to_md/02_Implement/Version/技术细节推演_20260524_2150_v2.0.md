## 0. 需求变更

**原目标**：MathType 公式 → 提取 WMF → 引用为 `![公式](rId.wmf)`

**新目标**：MathType 公式 → 提取 WMF → **OCR 识别为 LaTeX** → 内联 `$$...$$` 写入 MD

**原因**：目标 MD 由无视觉能力的 Agent 读取，图片引用不可用。公式必须是可解析的 LaTeX 文本。

## 1. MathType → LaTeX 技术方案调研

### 1.1 纯代码解析 MTEF 二进制 → LaTeX

| 方案 | 可行性 | 说明 |
|---|---|---|
| 手写 MTEF 解析器 | ❌ | MTEF 是 MathType 私有二进制协议，无公开规范 |
| MathType SDK (C#) | ❌ | 需要 MathType 7.x+ 商业许可证，闭源 DLL |
| Word COM 自动化 | ⚠️ | 需要 Word + MathType 插件同时安装，启动 Word 进程开销大 |

**结论**：纯 Python 解析不可行。走 OCR 图像识别路线。

### 1.2 OCR 图像识别路线

| 工具 | 类型 | 精度 | 延迟 | 依赖 |
|---|---|---|---|---|
| **pix2tex (LaTeX-OCR)** | 本地开源 | 高（印刷体） | ~1-2s/公式 | `pix2tex` + PyTorch (~500MB 模型) |
| **pix2text-mfr** | 本地开源 | 高（印刷体） | ~1s/公式 | `transformers` + ONNX |
| **Mathpix API** | 云端商业 | 极高 | ~2-3s/公式 | API Key + 网络 |

### 1.3 决策

**三级降级策略**：

| 优先级 | 方案 | 触发条件 |
|---|---|---|
| **1. pix2tex (LaTeX-OCR)** | `pip install pix2tex` → WMF→PNG → OCR → LaTeX | 默认首选 |
| **2. Mathpix API** | 设置 `--mathpix-key` 时启用 | pix2tex 不可用或用户显式指定 |
| **3. 占位标记** | `[公式: OCR 不可用]` | 全部失败时保留可见标记 |

## 2. WMF→PNG 转换修复

pix2tex 需要 PNG 输入。MathType 的 WMF 渲染图必须先转为 PNG。

**原 GDI 方案失败原因**：`ENHMETAHEADER` ctypes 结构体定义与 Windows API 不匹配，导致 `PlayEnhMetaFile` 崩溃。

**修正**：简化 GDI 调用：

```python
def wmf_to_png_simple(wmf_path, png_path):
    """使用 Windows GDI 简明渲染 WMF→PNG"""
    from PIL import Image
    # Pillow 在 Windows 上内置 WMF 支持 (GDI)
    try:
        img = Image.open(wmf_path)
        img.load()  # 触发 GDI 渲染
        img.save(png_path, "PNG")
        return png_path
    except OSError:
        return None
```

**关键**：Pillow 在 Windows 上通过 `Image.core.drawwmf()` 调用 GDI，**无需手动构造 ctypes 结构体**。之前过于复杂的 ENHMETAHEADER 方案反而引入了崩溃。仅在 `OSError: cannot render metafile` 时降级到 Inkscape。

## 3. 完整数据流（v2.0）

```
convert_docx_to_md(docx_path, ...)

Phase 1: 图片提取 + 关系映射
├── 解析 rels → {rId: target}
├── 解压 word/media/* 到 image_dir
├── 对每个 WMF: wmf_to_png_simple() → PNG
└── 构建 image_map: {rId: (png_path, is_formula)}

Phase 2: 文档遍历 (单轨 XML)
├── body.iterchildren() → w:p / w:tbl
├── w:r 内部:
│   ├── w:t → 纯文本
│   ├── w:drawing → rId → PNG → ![](path) (普通图片)
│   ├── w:object → rId → WMF → PNG → pix2tex OCR → $$latex$$
│   └── w:ins/w:del → 递归处理/跳过

Phase 3: 公式 OCR
├── 对每个 MathType 对应的 PNG:
│   ├── pix2tex model(img) → latex_string
│   ├── 缓存结果避免重复 OCR
│   └── 失败 → Mathpix API → [公式: OCR 失败]
└── 替换 MD 中的 ![公式](path) 为 $$latex$$

Phase 4: 输出
└── 写入 .md 文件
```

## 6. 公式 OCR 的预期效果与局限

- **印刷体数学公式**（MathType 产出）：pix2tex 精度 > 95%，常见符号全覆盖
- **复杂矩阵/多行公式**：可能需要 Mathpix API 兜底
- **内联小公式**（如 `$x^2$`）：pix2tex 处理良好
- **首次运行**：需下载 ~500MB 模型权重（自动缓存）
