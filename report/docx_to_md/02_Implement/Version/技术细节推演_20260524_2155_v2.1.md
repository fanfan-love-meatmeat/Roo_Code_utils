## 1. 三项工程优化验证

### ✅ 优化 1：并发 OCR + MD5 哈希缓存 — 确认必要

**检索结论**：
- PyTorch 官方论坛确认：`ThreadPoolExecutor` 用于 PyTorch 推理时需注意 GIL 问题，但由于 `pix2tex` 的 GPU 推理在实际 CUDA 操作期间会释放 GIL，线程池并发是有效的
- 200 个公式 × 2s/个 = 400s 单线程 → 4 线程并发 → ~100s，显著降低 Agent 超时风险
- MD5 哈希去重：学术论文中公式重复率通常 30%~50%（如上下文多次出现的 `$x$`、`$\theta$`、`$\sigma^2$` 等参数符号）

**实现方案**：

```python
import hashlib
from concurrent.futures import ThreadPoolExecutor, as_completed

def _ocr_formulas_batch(png_paths: list[str], max_workers: int = 4) -> dict[str, str]:
    """
    并发 OCR 一批公式图片，返回 {png_path: latex_string}
    内置 MD5 缓存避免重复 GPU 推理。
    """
    cache = {}          # {md5: latex}
    futures = {}        # {future: png_path}
    results = {}        # {png_path: latex}

    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        for png_path in png_paths:
            md5 = _file_md5(png_path)
            if md5 in cache:
                results[png_path] = cache[md5]
                continue
            futures[executor.submit(_ocr_single, png_path)] = png_path

        for future in as_completed(futures):
            png_path = futures[future]
            try:
                latex = future.result()
                md5 = _file_md5(png_path)
                cache[md5] = latex
                results[png_path] = latex
            except Exception as e:
                results[png_path] = f"[公式: OCR 失败 - {e}]"

    return results


def _file_md5(path: str) -> str:
    """快速计算文件 MD5 哈希"""
    h = hashlib.md5()
    with open(path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            h.update(chunk)
    return h.hexdigest()


def _ocr_single(png_path: str) -> str:
    """单个公式 OCR（由线程池调用）"""
    from pix2tex.cli import LatexOCR
    from PIL import Image

    model = _get_ocr_model()  # 全局单例延迟加载
    img = Image.open(png_path).convert("RGB")
    return model(img)
```

### ✅ 优化 2：跨平台 WMF→PNG 兜底 — 确认必要

**检索结论**：
- Pillow 官方文档：`WMF/EMF` 仅在 Windows 上可读（依赖 GDI `drawwmf`）
- Linux/macOS 需 Inkscape (`inkscape --export-type=png`) 或 LibreOffice (`soffice --headless --convert-to png`)
- 平台检测 + 三级降级是标准模式

**实现方案**：

```python
def wmf_to_png(wmf_path: str, png_path: str) -> bool:
    """
    WMF → PNG 跨平台转换。返回 True 表示成功。
    """
    import platform

    # Tier 1: Pillow (Windows GDI)
    if platform.system() == "Windows":
        try:
            from PIL import Image
            img = Image.open(wmf_path)
            img.load()
            img.save(png_path, "PNG")
            return True
        except Exception:
            pass

    # Tier 2: Inkscape (跨平台)
    try:
        import subprocess
        subprocess.run(
            ["inkscape", wmf_path, "--export-type=png",
             f"--export-filename={png_path}"],
            capture_output=True, check=True, timeout=15
        )
        return True
    except Exception:
        pass

    # Tier 3: LibreOffice (Linux)
    try:
        import subprocess
        out_dir = os.path.dirname(png_path)
        subprocess.run(
            ["soffice", "--headless", "--convert-to", "png",
             "--outdir", out_dir, wmf_path],
            capture_output=True, check=True, timeout=30
        )
        return True
    except Exception:
        return False
```

### ✅ 优化 3：LaTeX 输出清洗 — 确认必要

**检索结论**：
- pix2tex 官方文档示例显示模型输出纯 LaTeX（如 `E = mc^{2}`），不附带环境包裹
- 但社区反馈实际使用中模型有时会输出 `\begin{equation}...\end{equation}`、`\(...\)`、`\[...\]` 等
- 部分第三方封装（如 Pix2Text）和不同版本的模型行为不一致
- 防御性清洗是必要的最佳实践

**实现方案**：

```python
import re

def _clean_latex(raw: str, is_inline: bool = False) -> str:
    """
    清洗 pix2tex 输出的 LaTeX 字符串：
    1. 剥离冗余数学环境包裹 (\begin{equation}, \(, \[ 等)
    2. 统一由我们的 Markdown 控制 $ 或 $$ 包裹
    """
    s = raw.strip()

    # 剥离 \begin{equation}...\end{equation}
    s = re.sub(r'\\begin\{equation\*?\}', '', s)
    s = re.sub(r'\\end\{equation\*?\}', '', s)

    # 剥离 \begin{align}...\end{align} 等多行环境
    s = re.sub(r'\\begin\{align\*?\}.*?\\end\{align\*?\}', '', s, flags=re.DOTALL)

    # 剥离 \(...\) 和 \[...\] 包裹
    s = re.sub(r'^\\\(\s*', '', s)
    s = re.sub(r'\\\)\s*$', '', s)
    s = re.sub(r'^\\\[\s*', '', s)
    s = re.sub(r'\\\]\s*$', '', s)

    # 剥离 $$...$$ 包裹（避免双包裹）
    s = re.sub(r'^\$\$\s*', '', s)
    s = re.sub(r'\$\$\s*$', '', s)

    # 剥离 $...$ 包裹
    if s.count('$') >= 2:
        s = re.sub(r'^\$\s*', '', s)
        s = re.sub(r'\$\s*$', '', s)

    # 规范化空白
    s = re.sub(r'\s+', ' ', s).strip()

    return s
```

## 2. 最终架构（v2.1）

```
convert_docx_to_md(docx_path, ...)

Phase 1: 图片提取
├── rels 解析 → image + oleObject 类型
├── zipfile 解压 word/media/*
├── WMF → PNG (跨平台三级降级)
└── image_map: {rId: png_path}

Phase 2: 文档遍历 (含 AlternateContent / w:ins/w:del 修复)
├── para._p.iterchildren() → w:r/w:ins/w:del
├── w:r 内部 iterchildren → w:t / w:drawing / w:object / m:oMath
├── w:object → rId → 收集到公式列表 (待OCR)
├── w:drawing → rId → ![](path) (普通图片)
└── 生成中间 MD (公式位用 [OCR_PENDING:rId] 占位)

Phase 3: 并发公式 OCR (新增)
├── 收集所有公式 PNG 路径
├── ThreadPoolExecutor(max_workers=4) 并发
├── MD5 缓存 → 跳过重复公式
├── pix2tex → LaTeX → 清洗 → $$...$$
└── 失败 → Mathpix API (可选) → [公式: OCR 失败]

Phase 4: 输出
├── 替换占位符为 LaTeX
└── 写入 .md
```
