## 2. 修复方案

### 🔴 修复 1：OCR_BLOCK / OCR_INLINE 上下文占位符

**原问题**：`[OCR_PENDING:rId]` 不区分公式在表格内/外。Phase 3 统一包裹 `\n\n$$...$$\n\n`，表格内公式导致 pipe table 语法断裂。

**修复**：Phase 2 生成两种占位符：
- 表格外 → `[OCR_BLOCK:rId]` → Phase 3 替换为 `\n\n$$latex$$\n\n`
- 表格内 → `[OCR_INLINE:rId]` → Phase 3 替换为 `$latex$`

```python
# Phase 2: 区分上下文
tag = "OCR_INLINE" if is_in_table else "OCR_BLOCK"
result.append(f"[{tag}:{rid}]")

# Phase 3: 分别替换
md_content = re.sub(r'\[OCR_BLOCK:' + re.escape(rid) + r'\]',
                    f'\n\n$${latex}$$\n\n', md_content)
md_content = re.sub(r'\[OCR_INLINE:' + re.escape(rid) + r'\]',
                    f'${latex}$', md_content)
```

### 🟡 修复 2：Phase-2 感知的真分流（延后到 DOM 遍历）

**原问题**：Phase 1 按 `.wmf` 后缀盲目分配目录，无 DOM 上下文。

**修复**：
1. Phase 1：所有文件提取到 `image_dir/` 扁平目录（不分子目录）
2. Phase 2：遍历 DOM 时，收集 `formula_rids`（来自 `_is_mathtype_ole()` 判断）
3. Phase 2 之后：创建 `figures/` + `formulas/` 子目录，移动文件，更新 image_map

```python
# 后处理：真分流
for rid in formula_rids:
    if rid in image_map:
        old_path = image_map[rid]
        new_path = os.path.join(formulas_dir, os.path.basename(old_path))
        os.rename(old_path, new_path)
        image_map[rid] = new_path

# 剩余文件 → figures/
for rid, path in list(image_map.items()):
    if rid not in formula_rids and rid != "_base_dir":
        new_path = os.path.join(figures_dir, os.path.basename(path))
        os.rename(path, new_path)
        image_map[rid] = new_path
```

### 🟠 修复 3：平台熔断 — Windows GDI 强绑定

**原问题**：Linux 环境 Pillow 无法渲染 WMF（`OSError: cannot render metafile`），Inkscape/LibreOffice 降级逻辑徒增维护负担。

**检索验证**：Pillow 官方文档确认 `WMF/EMF` 仅在 Windows 上可读（依赖 `Image.core.drawwmf()` GDI 调用）。Stack Overflow 多例确认 Linux 环境抛出 `OSError`。

**修复**：删除三级降级，改为 Fail-Fast：

```python
def wmf_to_png(wmf_path: str, png_path: str) -> bool:
    import platform
    if platform.system() != "Windows":
        raise RuntimeError(
            "WMF/MathType 渲染强依赖 Windows GDI。请在 Windows 环境下运行。"
        )
    from PIL import Image
    img = Image.open(wmf_path)
    img.load()
    img.save(png_path, "PNG")
    return True
```
