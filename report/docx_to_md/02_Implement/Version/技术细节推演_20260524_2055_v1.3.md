## 1. v1.3 新增修正：Markdown 表格与块级公式的语法冲突

### 验证结果

- Hugo/Goldmark 官方文档明确声明：**"Block-level elements cannot be inserted in a table"** — `$$...$$` 作为块级元素，直接放入 pipe table 单元格会导致整表渲染断裂
- Obsidian 论坛大量实战反馈：`$$` 块公式 + `<br>` 在表格内导致渲染异常
- CommonMark 社区讨论确认：pipe table 只支持行内级（span-level）内容
- 标准解法：a) 使用 `<br>` 替代换行，b) 块级公式降级为行内 `$...$` 语法

### 修正方案

在 `parse_paragraph_children()` 增加 `is_in_table=False` 上下文参数：

```python
def parse_paragraph_children(para, image_map: dict, is_in_table: bool = False) -> str:
    for child in para._p.iterchildren():
        tag = child.tag.split('}')[-1] if '}' in child.tag else child.tag
        
        if tag == 'r':
            yield extract_run_text(child)
        elif tag == 'oMath':
            latex = omml_to_latex(child)
            yield f'$${latex}$$' if not is_in_table else f'${latex}$'
        elif tag == 'oMathPara':
            latex = omml_to_latex(child)
            # 块公式在表格内 → 降级为行内 + <br> 替换换行
            if not is_in_table:
                yield f'$${latex}$$'
            else:
                yield f'${latex.replace(chr(10), " <br> ")}$'
        elif tag == 'drawing':
            rId = extract_blip_rid(child)
            yield f'![]({image_map[rId]})'
        elif tag == 'object':
            rId = extract_ole_image_rid(child)
            yield f'![公式]({image_map[rId]})'
```

表格单元格内调用时传入 `is_in_table=True`：

```python
def parse_table(table_element, image_map: dict) -> str:
    rows = []
    for tr in table_element.iterchildren():
        # ...
        for tc in tr:
            cell_texts = []

```
