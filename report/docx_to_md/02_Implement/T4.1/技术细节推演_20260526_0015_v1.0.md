## 1. 检索与论证

### 1.1 Wiris 官方证据（四类记录）

| 记录类型 | Tag 字节 | 下一字节 | 再下一字节 | 来源行号 |
|----------|----------|----------|------------|----------|
| **CHAR** | `02` | `00` (options) | `88` (typeface) | 0x0100 |
| **LINE** | `01` | `00` (options) | — | 0x00C1 |
| **TMPL** | `03` | `00` (options) | `1C` (selector) | 0x00E7 |
| **EQN_PREFS** | `12` | `00` (options) | `08` (count) | 0x0056 |

**结论**: MTEF v5 协议中，**options 是独立字节**，紧接在 tag 之后。`tag & 0x0F` 和 `tag >> 4` 均不对应协议定义。

### 1.2 当前代码的协议偏差

```python
# 当前 (错误): 假设 tag 字节高4位=options
tag = self._read_byte()
record_type = tag & 0x0F     # 对标准记录来说 tag==record_type，但语义不准确
options = tag >> 4           # ❌ 协议中不存在此编码
```

正确协议：
```
tag(1 byte) → options(1 byte) → payload(variable)
```

### 1.3 CTO 指令评估

| 指令 | 与协议一致性 | 评估 |
|------|-------------|------|
| 指令一：双字节包头 | ✅ 完全一致 | 经 Wiris 四类记录独立验证 |
| 指令二：删除 typeface=0 重对齐 | ✅ 正确 | 旧补丁是为掩码错误擦屁股，步长修正后不再需要 |
| 指令三：闭眼消费/信仰之跃 | ✅ 防御 | 步长严丝合缝后，前瞻预读不再必要 |

## 2. 修改清单

### 2.1 涉及函数

| 方法 | 变更类型 | 说明 |
|------|----------|------|
| `_parse_line()` | **重构** | `tag = _read_byte()` → `options = _read_byte()` 替换 `tag>>4` |
| `_consume_size_prefix()` | **重构** | 同上，双字节读取 |
| `_read_char_value()` | **删除** | 移除 `typeface == 0` 重对齐补丁 |
| `_skip_header()` | **微调** | 扩展记录保持 `full tag` 判断；标准记录新增 options 跳过 |

### 2.2 关键变更

```python
# _parse_line — 所有标准记录节点
tag = self._read_byte()
options = self._read_byte()
record_type = tag               # tag 值即 record_type (0x00-0x0F)

# CHAR 分支 — 移除重对齐
# 删除: elif typeface == 0 and self.data[self.pos] >= 128: ...
# 改为纯粹的单/双字节读取

# _consume_size_prefix — 双字节读取
tag = self._read_byte()
options = self._read_byte()     # 忽略，SIZE 记录的 options 在此阶段不重要
if tag == SIZE: ...
elif tag in (FULL, SUB, ...): ...
elif tag in (LINE, CHAR, TMPL, END): return
```

### 2.3 游标前进路径对比

**修复前** (`02 04 84 B8`):
```
read tag=02 → options=02>>4=0 → typeface=04 → char_val=0xB884(uint16) → '뢄'
```

**修复后** (`02 04 84 B8`):
```
read tag=02 → read options=04 → typeface=84 ≥128 → MTCode单字节=B8 → MTCode映射 ✅
```

## 3. 风险矩阵

| 风险 | 概率 | 缓解 |
|------|------|------|
| `_skip_header` 中标准记录需读取 options 后跳变 | 低 | 扩展记录不受影响；标准记录在 header 循环中 `self.pos += 1` 改为 `self.pos += 2` |
| 删除重对齐后公式回归 | 中 | 双字节包头从根本上解决错位，typeface=0 场景自然消失 |
| `_consume_size_prefix` 正确性 | 低 | SIZE 记录的 options 可忽略，数据段由 `_parse_size_record` 处理 |
