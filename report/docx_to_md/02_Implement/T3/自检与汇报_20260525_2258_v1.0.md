## 2. 架构安全自检

### 2.1 防御机制验证

| 防御层 | 状态 | 说明 |
|--------|------|------|
| EQN_PREFS options 字节 | ✅ 已修复 | 原先遗漏导致 size_count 从 0x00(错误) → 0x08(正确) |
| Nibble 数组自校准搜索 | ✅ 已实装 | 逐字节扫描 + `idx > 10` 严格校验，拒绝将 nibble 字节误判为 style_count |
| Style 定义 uint8 消费 | ✅ 已实装 | 与 Wiris 规范一致 |
| Pre-AST SIZE 前缀 | ✅ 已实装 | `_consume_size_prefix` 消费 SIZE/FULL/SUB/SUB2/SYM/SUBSYM |
| CHAR 防污染断言 | ✅ 已实装 | 控制字符/私用区 → debug log |

### 2.2 代码扁平化

- `_parse_eqn_prefs`：从 1 行硬编码 skip → 3 步骤清晰分段（options → nibble_search → style_consume）
- `_consume_dimension_array`：自校准搜索在一层循环内完成，无嵌套
- `_consume_style_definitions`：简单 for 循环，提前 break

### 2.3 侧面破坏风险

| 风险项 | 评估 |
|--------|------|
| 自校准搜索误判 | 低 — 已加入 `idx > 10` 校验，仅 3/263 公式残留误判 |
| Style 定义格式假设 (uint8) | 低 — Wiris 规范的 "unsigned integer" 以 uint8 实现，仅当索引 ≥ 256 时需调整为 uint16 |
| Pre-AST 消费 SIZE 后 cursor 跳跃 | 安全 — 有 `max_steps=20` 上限 |

### 3.2 输出质量演进

| 阶段 | 典型输出 | 问题 |
|------|----------|------|

### 3.3 残留 3 公式分析

3 个公式（oleObject21/107/194, 均为 292B）输出 `⢂` (U+2882)。这些公式的 nibble 数组长度可能不同，自校准搜索未找到正确边界。建议后续 Sprint 处理。
