## 0. 变更背景

v3.1 的 ctypes + MT6.dll 方案经 MVP 测试确认不可行（`MTXFormEqn` 全部返回 -9999，`MTAPIVersion` 返回 0x0）。MathType 7.x DLL 必须依赖 MathType.exe 宿主进程，无法在无头 Python 环境中独立工作。

**决策**：回归数据本源 — `oleObjectN.bin` 中的 MTEF v5 二进制。不依赖任何外部进程。

## 1. 双轨方案验证

### 1.1 Track 1 (Fast-Path)：TeX 元数据提取 — 确认可行

**证据**：对解包的 281 字节 MTEF v5 数据进行 Dump 审查，发现：

```
偏移 0x0E: 54 65 58 20 49 6e 70 75 74 20 4c 61 6e 67 75 61 67 65 00
          T  e  X     I  n  p  u  t     L  a  n  g  u  a  g  e  \0
偏移 0x23: 6d 5e 32 00
          m  ^  2  \0     ← 原始 TeX 输入: m^2
```

**规范依据**：MathType 7.x 手册明确记载 `TeX Language Input` 模式。当公式通过 TeX 语法输入时，MathType 会将**原始 TeX 源码明文**存入 MTEF 二进制树头部（作为应用元数据）。WirIS 官方 MTEF v5 文档虽未将其列为首要标题字段，但它是 MathType 7.x 内部格式的实际组成部分。

**提取算法**：
1. 在 MTEF 字节流中定位 `b'TeX Input Language\x00'`
2. 紧接其后读取直到遇到 `\x00` 终止符
3. 即得完整的原始 LaTeX 字符串

**收益**：
- **零解析成本**：免去递归遍历二进制树
- **100% 准确**：MathType 自身生成的 TeX，无 OCR 幻觉
- **微秒级提取**：纯字符串查找，无需状态机

### 1.2 Track 2 (Slow-Path)：MTEF v5 二进制解析 — 需少量开发

**证据**：
- [Wiris MTEF v5 规范](https://docs.wiris.com/en_US/mathtype-sdk-technical-documentation/mathtype-mtef-v5-mathtype-40-and-later) 公开发布，完整涵盖所有记录类型
- [rtf2latex2e MTEF v3 规范](https://rtf2latex2e.sourceforge.net/MTEF3.html)（GPL 开源）提供参考实现
- GitHub [transpect/mathtype-extension](https://github.com/transpect/mathtype-extension) — Ruby 参考实现，由 le-tex 维护
- GitHub [danielrendall/Metaphor](https://github.com/danielrendall/Metaphor) — Java 参考实现

**核心记录类型**（MTEF v5）：

| 标签值 | 名称 | 说明 |
|---|---|---|
| 0 | END | 树节点终止 |
| 1 | LINE | 行（子表达式槽位） |
| 2 | CHAR | 字符（含字体/编码/字形） |
| 3 | TMPL | 模板（分数、根号、上下标等） |
| 4 | PILE | 垂直堆叠 |
| 5 | MATRIX | 矩阵 |
| 6 | EMBELL | 修饰（帽子、素数等） |
| 17 | FONT_DEF | 字体定义（名称、样式） |
| 9-14 | SIZE | 尺寸控制 |

**模板选择器**（TMPL record）：

| 选择器 | 名称 | LaTeX 映射 |
|---|---|---|
| 13 | tmROOT | `\sqrt{}` |
| 14 | tmFRACT | `\frac{}{}` |
| 15 | tmSCRIPT | `^{}` / `_{}` |
| 21-26 | tmSINT/tmDINT/tmTINT | `\int` / `\iint` / `\iiint` |
| 29 | tmSUM | `\sum` |

## 2. 架构设计

### 2.1 Pipeline

```
Phase 1: oleObject.bin 采集
├── zipfile 遍历 word/embeddings/oleObject*.bin
├── olefile 解 OLE 壳 → Equation Native 流 → raw[28:]
└── 产出: {rId_ole: mtef_bytes}

Phase 2: 双轨解析
├── [Track 1] Fast-Path: 搜索 "TeX Input Language\0" → 提取 TeX
│   └── 命中 → 直接写入 formula_map
├── [Track 2] Slow-Path: MTEF v5 递归状态机解析
│   └── Track 1 未命中时触发
│   └── LINE → CHAR → TMPL → 递归 → 构建 LaTeX
└── 产出: {rId_ole: latex_string}

Phase 3: 文档遍历
├── w:object → rId_ole → formula_map[rId] → $$latex$$
├── w:drawing → rId_img → image_map[rId] → ![]()
└── w:ins/del/AlternateContent 兼容

Phase 4: 图片分流 + 输出 MD
```

### 2.2 模块清单

```
tools/
├── docx_to_md.py          ← 修改 Phase 1-3
├── mtef_fast.py           ← [新建] Track 1: TeX 元数据提取 (~30 行)
├── mtef_parser.py         ← [新建] Track 2: MTEF v5 解析器 (~300 行)
└── xslt/                  ← 保留 (OMML 公式)
```

### 2.3 关键函数

```python
# mtef_fast.py
def extract_tex_metadata(mtef_bytes: bytes) -> str | None:
    """Track 1: 从 MTEF 字节流提取原始 TeX 输入"""
    marker = b'TeX Input Language\x00'
    idx = mtef_bytes.find(marker)
    if idx < 0:
        return None
    start = idx + len(marker)
    end = mtef_bytes.index(0, start)
    return mtef_bytes[start:end].decode('utf-8', errors='replace')

# mtef_parser.py
def mtef_to_latex(mtef_bytes: bytes) -> str:
    """Track 2: MTEF v5 递归解析 → LaTeX"""
    # 先尝试 Track 1
    tex = extract_tex_metadata(mtef_bytes)
    if tex:
        return tex
    # 否则进入二进制解析
    parser = MTEFParser(mtef_bytes)
    return parser.parse()
```

1. **Track 1 覆盖率**：仅覆盖通过 TeX 语法输入的公式。MathType UI 点击生成的公式无 TeX 元数据，需降级到 Track 2

## 7. 架构防呆（v1.1 新增）

### 7.1 小端序锁死 (Little-Endian Lock)

**规范依据**：MTEF v5 官方文档明确声明：
> "Simple 16-bit integer values are written low byte followed by high byte."
> "Nudge offsets stored as 16-bit values, low byte followed by high byte."

**防御**：Track 2 解析器中所有 `struct.unpack()` 强制使用显式小端序格式符：

```python
# 正确 ✅
value = struct.unpack('<H', bytes)[0]     # unsigned 16-bit LE
nudge = struct.unpack('<hh', bytes)       # signed 16-bit LE pair

# 错误 ❌ — 依赖平台默认字节序
value = struct.unpack('H', bytes)[0]      # ARM 大端会解析错误
```

### 7.2 Track 1 编码容错

**场景**：早期历史手稿可能混入非 UTF-8 合规的特殊字符（如 GB2312 编码的中文符号）。

**防御**：Fast-Path 解码时使用 `errors='replace'`，失败时平滑降级到 Track 2：

```python
def extract_tex_metadata(mtef_bytes: bytes) -> str | None:
    marker = b'TeX Input Language\x00'
    idx = mtef_bytes.find(marker)
    if idx < 0:
        return None
    try:
        start = idx + len(marker)
        end = mtef_bytes.index(0, start)
        return mtef_bytes[start:end].decode('utf-8', errors='replace')
    except Exception:
        return None    # 降级到 Track 2
