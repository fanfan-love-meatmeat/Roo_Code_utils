## 3. 架构安全自检

### 3.1 防御机制

| 防御层 | 状态 | 说明 |
|--------|------|------|
| typeface≥128 门控 | ✅ | 强制单字节 MTCode，阻断双字节→CJK 路径 |
| typeface=0 重对齐 | ✅ | 窥探后字节≥128 → 自动重解释为 MTCode |
| 字体路由 | ✅ | Symbol/MT Extra/Text 三表分发 |
| 未命中占位 | ✅ | `[MTCode:0xNN]` 替代生僻 CJK |
| PUA 监控 | ✅ | `_char_to_latex` 保留 chr() + debug log |

### 3.2 残留问题

| 项目 | 说明 |
|------|------|
| `o` vs `x` | MTCode 0x6F 在 Symbol 表中映射为 `o`，需确认文档中该变量名是否为 `o` |
| 下标结构 | TMPL(subscript) 模板未在输出中显示（如 `x_c` 应输出 `x_{c}`），需进一步调试模板展开逻辑 |
