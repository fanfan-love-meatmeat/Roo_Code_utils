# 技术细节推演：T4.2 追加 — `\xi` 映射崩塌与 Inline 判定

**日期**: 2026-05-26 01:08  
**版本**: v2.0（追加）  
**依据**: 端到端输出 + 公式 #74-#78 Hex Dump

---

## 1. 异常一：`\xi` 下标映射崩塌

### 1.1 现象

COCO 关键点坐标公式输出：
```
$$_{K_{1}}\xi$$   ← 应为 $$K_{10}$$
$$_{K_{9}}$$      ← 应为 $$K_{9}$$
```

### 1.2 Hex Dump 证据（公式 #74, `_{K_{1}}\xi`）

**方程数据**（pos 217, 20B）:
```
01 00 03 00 01 03 00 01 00 10 00 00 00 00 00 00 00 0F 01 02
```

**MTEF 头部**（前 20B）:
```
05 01 00 07 04 44 53 4D 54 37 00 00 ...
```
`eq_opt = 0x00`（bit0=0 → Display 标记）

### 1.3 根因分析

当前解析路径：
```
LINE(01,00)
  └─ TMPL(03, nibble opts=0) → selector=00 → subscript route (T4.2)
       └─ 跳过前导 END(00)
       └─ LINE(01,00)
            └─ TMPL(03, nibble opts=0) → subscript route → base=K → K_1
            └─ ... (嵌套模板后)
       └─ 返回: "_{K_{1}}"  (base 为空 + subscript)
  └─ CHAR(tf=03, mtcode=78) → '\\xi'  ← 父级继续解析的残留 CHAR
```

**核心问题**: 模板 subscript（selector=0x00）的 `_parse_line()` 在消费到 `K_{1}` 后提前终止，未将后续的 CHAR `\xi` 纳入 subscript 子 LINE。导致 `\xi` 作为独立父级 CHAR 输出。

**字节级根因**: `01 00 10 00 00 00 00 00 00 00 0F 01 02`
- `10` 被 two-byte header 读作 `tag=0x10`(COLOR_DEF) → `_parse_color_def()` 消费 8 字节 → 越界吞掉后续真实数据

COLOR_DEF(0x10) 是扩展记录，不应出现在方程 AST 中。它出现在此处说明要么是 nibble-packed（0x10 应解读为其他 tag），要么 `_skip_header` 未完全消费元数据。

### 1.4 修复方案

| 方案 | 内容 |
|------|------|
| A | `_parse_line` 中对 `tag==COLOR_DEF` 改为仅跳过 1 字节（two-byte header 中已消费 options，剩余 8 字节数据一次性跳过有风险） |
| B | 确认 subscript 模板的 `_parse_line()` 正确消费所有子 LINE，不在未遇到 END 时提前退出 |
| C | COLOR_DEF 在 `_parse_line` 中降级为未知记录跳过 |

## 2. 异常二：Inline/Display 判定

### 2.1 现象

关键点公式（如 `K_7`, `K_9`）在 MD 中以 `$$...$$` 渲染，撕裂句子连贯性。用户期望为行内 `$...$`。

### 2.2 Hex 证据

公式 #56, #57, #74-#78 等全部 `eq_opt=0x00`（bit0=0）：
```
mtef_head: 05 01 00 07 04 44 53 4D 54 37 00 00 ...
                                                 ↑ eq_opt=0x00
```

`is_inline_equation()` 返回 `False` → 渲染为 `$$`。

### 2.3 根因

**非解析器 Bug。** MTEF 二进制流中 equation options 字节 bit0=0，表明 MathType 在保存时将公式写为 **Display（独立公式）**。这是源 DOCX 的格式属性，解析器正确反映了物理字节。

### 2.4 修复建议

- **短期**: 无需修改解析器。该行为与 MTEF 数据一致。
- **长期**: 可增加启发式推测：若公式内容在文本中仅占 1-2 字符且 inline 标记更符合常见排版，可 override。但这是格式推测，非协议保证。
