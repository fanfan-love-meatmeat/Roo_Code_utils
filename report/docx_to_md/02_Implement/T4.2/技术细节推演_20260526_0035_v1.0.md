## 1. 指令一：MTCode 占位符映射

### 1.1 现状

| MTCode | 当前输出 | 出现位置 | 预期含义 |
|--------|----------|----------|----------|
| `0xB8` | `[MTCode:0xB8]` | S3/S4 俯仰角 | `\theta_p` |
| `0xD5` | `[MTCode:0xD5]` | S3/S4 偏航角 | `\theta_y` |

### 1.2 修复

直接补入 `_MTOKEN_MT_EXTRA` 字典（或更合适的 Symbol 映射表，取决于这些字符在 MathType 中的实际字体归属）。

**方案**: 在 `_MTOKEN_MT_EXTRA` 末尾追加：
```python
0xB8: '\\theta_p',   # 俯仰角
0xD5: '\\theta_y',   # 偏航角
```

由于这两个 MTCode 值通过 `_mtcode_to_latex` → `_MTOKEN_MT_EXTRA.get()` 路由，添加后 `[MTCode:0xB8]` 将直接输出 `\theta_p`。

## 2. 指令二：`\langle\rangle` 拓扑塌缩修复

### 2.1 现状

输出中出现 `\Delta\langle\rangle`、`P\langle\rangle` 等，`\langle\rangle` 为空内容。期望输出为 `\Delta_{c}`、`P_{center}` 等下标结构。

### 2.2 根因分析

公式 #0 的数据中 TMPL 记录为 `03 00 01 00`：
- `03` = TMPL tag
- `00` = **selector = 0x00** → `TMPL_SELECTORS[0x00]` = `('angle', 'angle')` → 输出 `\langle\rangle`
- `01` = variation
- `00` = tmpl_opts

当前 `_template_to_latex` 对 selector=0x00/0x01 的处理：
```python
elif selector in (0x00, 0x01):  # fences → 尖括号/圆括号
    fencetype = TMPL_SELECTORS.get(selector, ('paren', ''))[0]
    f = fences.get(fencetype, ('(', ')'))
    arg = ''.join(subs)
    return f'{f[0]}{arg}{f[1]}'
```

selector=0x00 → angle → `\langle\rangle`（子 LINE 为空，arg=''）。

### 2.3 拐点分析

模板的 `variation=1` 且 `subs` 为空，而 base 字符在 Parent Context Callback 中已被正确 `parts.pop()`（所以输出是 `\Delta\langle\rangle` 而非 `\langle\rangle\Delta`）。这说明：
- **Parent Context 正确** — base 已弹出
- **模板路由错误** — selector=0x00 被路由到 angle fence，而实际应路由到 subscript

### 2.4 修复

**Step 1**: 打印捕获生成 `\langle\rangle` 时的 `selector` 和 `variation` 值，确认具体字节。

**Step 2**: 对于 `selector=0x00, variation=1` 的疑似下标变体，在 `_template_to_latex` 的 fence 分支前插入路由：
```python
elif selector == 0x00 and variation == 1:
    # selector=0x00 + var=1 → 私有下标变体
    return f'{base}_{{{subs[0] if subs else ""}}}'
```

**Step 3**: 子节点容错 — `subs` 为空时 `_{}` 会产生 LaTeX 语法错误。改为 `_{}` 不输出或输出 `_{}` with empty brace。

### 2.5 备选方案

若 `variation=1` 不是下标信号，则打印 `selector` 真实值后按指令将其加入 `TMPL_SELECTORS` 并路由到 subscript 分支。
