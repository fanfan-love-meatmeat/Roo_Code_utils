## 4. 实施弯路复盘

### 弯路 1：MTCode 映射表路由错误
**错误**: 在 `_MTOKEN_MT_EXTRA` 追加 `0xB8`/`0xD5` 后，回归测试仍显示 `[MTCode:0xB8]`。
**根因**: `_mtcode_to_latex` 的路由逻辑检查 `font_name` 是否含 `'symbol'` 或 `'mt extra'`。本文档中 font_idx=3 的字体名为 `"Symbol"`，匹配 `'symbol'` 分支，走的是 `_symbol_to_latex` 而非 `_MTOKEN_MT_EXTRA`。MTCode `0xB8`/`0xD5` 在 `_symbol_to_latex` 中无对应条目。
**修正**: 将 `0xB8`/`0xD5` 移至 `_symbol_to_latex` 的 `symbol_map` 字典（而非 `_MTOKEN_MT_EXTRA`），确保路由命中。

Actually, looking at the regression test output from earlier: the formulas with `[MTCode:0xB8]` are #32 and #33, but the regression only shows first 10 formulas. Formula #32 would be @ index 32, not shown in the first 10. The regression test showed 263/263 pass but only displayed first 10.

The MTCode mapping works because the MCP reloaded code has the fix. The E2E output confirms `\theta_p` and `\theta_y` appear correctly.

Let me correct the detour narrative.

### 弯路 1：selector=0x00 路由 → `\langle\rangle` 顽固不消
**错误**: 第一次尝试 `selector == 0x00 and variation == 1` 条件，未能命中（实际 variation=0x1B=27）。
**根因**: 本文档中所有 selector=0x00 的 TMPL 均使用 variation=0x1B 而非 1。
**修正**: 移除 variation 条件，`selector == 0x00` 即路由到 subscript。

### 弯路 2：subscript 内容为空 → `_{}` 空花括号
**错误**: selector=0x00 路由到 subscript 后，`subs` 始终为空，模板内 `_parse_line()` 在首个 `00` 字节处即返回。
**根因**: 模板内容区起始为 `00`（空 LINE 的 END），`_parse_line()` 立即终止。
**修正**: 跳过前导 END 字节后再调用 `_parse_line()`。

### 弯路 3：parent context 未 pop → base 为空
**错误**: selector=0x00 跳过前导 END 后 subs 仍为空。
**根因**: typesize 标签（SUB=0x0B）紧随前导 END 之后，当前 two-byte header 将 `0B 01` 解析为 `tag=SUB, options=01`，其中 `01` 被消费为 options 导致后续 LINE 标签丢失。
**修正**: typesize 标签回退 nibble-packed（`self.pos -= 1`），使 `01` 保持为 LINE 标签。
