**依据**: 代码审查 + MTEF v5 规范 TMPL 节点结构  
**版本**: v1.0

## 1. Bug 定位

### 1.1 当前代码的结构性错误

`_template_to_latex` (L537-595) 对 subscript/superscript 的处理：

```python
elif selector == 0x0F:  # sub/superscript
    subs = self._parse_line()          # ← 递归解析模板子 LINE
    base = subs[0] if len(subs) > 0 else ''    # ← BUG! 这是下标内容,不是base!
    sub = subs[1] if len(subs) > 1 else ''
    return f'{base}_{{{sub}}}'
```

**问题**: `_parse_line()` 在 TMPL 内部被调用时，解析的是模板的**子 LINE 节点**（即下标/上标的内容）。`subs[0]` 是下标字符（如 `c`），**不是** base 字符（如 `x`）。

### 1.2 MTEF 结构示意

`x_c` 的 MTEF 编码：
```
LINE                          ← 父 LINE
  CHAR('x')                   ← base 字符，已在父 LINE 的 parts 中
  TMPL(subscript, var=1)      ← 模板节点
    LINE                      ← 子 LINE（下标内容）
      CHAR('c')               ← 下标字符
    END
  END
END
```

当前解析流程：
1. `_parse_line` → CHAR('x') → `parts.append('x')` ✅
2. `_parse_line` → TMPL → `_template_to_latex` 调用 `_parse_line`
3. 内层 `_parse_line` → LINE → CHAR('c') → 返回 `['c']`
4. `_template_to_latex`: `base = subs[0] = 'c'`, `sub = subs[1] = ''`
5. 返回 `{c}_{{}}` → `parts.append('{c}_{{}}')` ❌
6. 最终 `parts = ['x', '{c}_{{}}']` → 输出 `x{c}_{{}}` → 完全错误！

**正确流程**应生成 `{x}_{{c}}` → 输出 `x_{c}`。

### 1.3 影响范围

此 Bug 影响所有 subscript/superscript/fraction/radical 等需要**parent context binding** 的模板。当前输出：

| 期望 | 实际输出 |
|------|----------|
| `x_c` | `x{c}_{{}}` 或 `\Chi\nabla{}` |
| `\Theta_{hip}` | `\langle\rangle\Theta` |
| `\frac{a}{b}` | 分式子节点可能丢失 |

## 2. 重构方案

### 2.1 核心思路

引入 **Parent Context Callback** 机制：`_template_to_latex` 不再从 `subs` 提取 base，而是**从父 LINE 的 parts 栈中取回最后一个 CHAR 作为 base**。

### 2.2 修改清单

| 修改 | 方法 | 内容 |
|------|------|------|
| A | `_parse_line` → TMPL 分支 | 弹出 `parts[-1]` 作为 base，传入 `_template_to_latex` |
| B | `_template_to_latex` | 接收 `base` 参数（可选），subscript/superscript 从参数取 base |
| C | `_template_to_latex` | fraction/radical 保持子 LINE 自洽（无需 parent context） |
| D | `_parse_line` → TMPL 分支 | 未知模板降级为 `\boxed{...}` 并包裹子节点 |

### 2.3 关键逻辑变更

```python
# _parse_line 中 TMPL 分支
elif record_type == TMPL:
    if options & xfLMOVE:
        self._read_nudge()
    selector = self._read_byte()
    variation = self._read_byte()
    tmpl_opts = self._read_byte()

    # 需要 parent context 的模板（subscript/superscript/big_ops）
    if selector in (0x0F, 0x15, 0x16, 0x17, 0x18, 0x19, 0x21, 0x22, 0x23):
        base = parts.pop() if parts else ''
    else:
        base = ''

    latex = self._template_to_latex(selector, variation, tmpl_opts, base)
    parts.append(latex)

# _template_to_latex 中 subscript 分支
elif selector == 0x0F:  # sub/superscript
    subs = self._parse_line()   # 解析子 LINE → 下标/上标内容
    if variation == 0:          # superscript only
        return f'{base}^{{{subs[0] if subs else ""}}}'
    elif variation == 1:        # subscript only
        return f'{base}_{{{subs[0] if subs else ""}}}'
    elif variation == 2:        # both
        sub = subs[0] if len(subs) > 0 else ''
        sup = subs[1] if len(subs) > 1 else ''
        return f'{base}_{{{sub}}}^{{{sup}}}'
    return base
```

### 2.4 深度控制与未知模板降级

def _parse_line(self) -> list:
    self._depth += 1

    try:
        ...  # 主逻辑
    finally:
        self._depth -= 1

# 未知模板降级
else:  # _template_to_latex 的 else 分支
    subs = self._parse_line()
    arg = ''.join(subs) if subs else ''
