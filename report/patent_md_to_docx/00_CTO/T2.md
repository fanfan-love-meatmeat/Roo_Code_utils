# Task 2: 排版引擎视觉降维与合规性修正

> **执行人**: Python 开发团队
> **目标模块**: `tools/patent_style_utils.py` & `tools/patent_md_to_docx.py`
> **背景**: 当前 v1.0 版本的 DOCX 引擎在 H1/H2 标题对齐方式、下划线修饰以及数学公式占位符上，存在“过度工程”和“不符合 CNIPA 视觉习惯”的问题。本迭代需做精准减法与属性修正。

## Action Item 1: H1 一级标题的下划线与字体修正
* **定位**: `patent_md_to_docx.py` 主循环中处理 `level == 1` 的逻辑，或对应的 `add_section_title` 函数。
* **修改要求**:
  1. **开启下划线**：给承载 H1 标题的 `Run` 对象强行开启单下划线属性（`run.underline = True`）。
  2. **字体变更**：将中文字体从“宋体”改为“黑体”（或“楷体_GB2312”，视环境系统字体库而定），字号保持小二 (Pt(18))，保持居中对齐。

## Action Item 2: H2 二级标题的“左侧顶格”修正
* **定位**: `patent_md_to_docx.py` 主循环中处理 `level == 2` 的逻辑（即“技术领域”、“背景技术”等）。
* **修改要求**:
  1. **对齐方式变更**：将段落对齐方式从 `CENTER`（居中）强制改为 `LEFT`（左对齐）。
  2. **强制无缩进**：显式声明该段落首行缩进为 0 (`paragraph_format.first_line_indent = None` 或 `0`)，确保绝对顶格。
  3. **字体设定**：中文字体设为“黑体”，加粗，字号四号 (Pt(14))。

## Action Item 3: 数学公式的“绝对纯净度”降维 (为 MathType 宏让路)
* **定位 1 (块级公式)**: `add_display_formula` 函数。
  * **修改要求**：彻底删除注入灰色底纹（`w:shd`）和深红色字体（`RGBColor(0xC0,0x00,0x00)`）的代码。只保留居中对齐、段前段后间距（Pt(6)），将公式原始字符串（带 `$$`）作为普通文本写入（Times New Roman, Pt(12)）。
* **定位 2 (行内公式)**: `add_rich_paragraph` 函数中的 `text` 解析节点。
  * **修改要求**：**彻底删掉**原有的针对 `$` 符号的 `re.split` 拆分与红色高亮逻辑。直接将 Markdown AST 解析出的纯文本 `txt.content` 原样作为 `Run` 写入。确保 `$` 与普通文本在同一个物理 Run 对象中不断裂。

## 验收标准 (Acceptance Criteria)
1. 生成的 DOCX 中，“权利要求书”、“说明书”等大标题底部必须出现单下划线。
2. “技术领域”、“背景技术”必须靠左侧绝对顶格，不随正文进行 2 字符缩进。
3. 文档中所有的 `$l$`、`$R_t$` 等公式标记必须是标准黑色字体，且按下 `Alt + \` (MathType 快捷键) 时能够成功被宏捕获并转换为公式实体。
