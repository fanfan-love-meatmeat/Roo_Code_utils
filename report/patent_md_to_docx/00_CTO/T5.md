# Task 5: 终极排版体验优化 — 原生分页与格式隔离

> **执行人**: Python 开发团队
> **目标模块**: `tools/patent_style_utils.py` & `tools/patent_md_to_docx.py`
> **背景**: 使用物理 `doc.add_page_break()` 会在文档中留下硬分页符，用户手动删除时会导致 H1 标题倒吸至上一页。同时，H1 标题带有段落边框，若不追加空段落，用户在末尾粘贴图片时会继承边框与 18pt 字体。本迭代需利用原生段落属性解决此交互痛点。

## Action Item 1: H1 启用原生“段落前分页” (Page Break Before)
* **定位**: `patent_style_utils.py` -> `add_section_title()`
* **修改要求**: 为所有 H1 段落原生开启 `page_break_before` 属性。让 Word 引擎自动处理五大书的分页，彻底抛弃硬分页符字符。（注：Word 会自动忽略文档绝对首段的该属性，所以首页不会出现空白页，非常安全）。
* **代码指引**:
  ```python
  def add_section_title(doc, text: str):
      p = doc.add_paragraph()
      p.alignment = WD_ALIGN_PARAGRAPH.CENTER
      p.paragraph_format.space_before = Pt(12)
      p.paragraph_format.space_after = Pt(12)
      
      # 【核心修正 1】：利用原生段落属性分页，杜绝硬分页符
      p.paragraph_format.page_break_before = True 
      
      # ... [保留 T4 的 3 空格清洗和底边框注入逻辑] ...
      
      return p
  ```

## Action Item 2: 移除硬分页与追加纯净“图片着陆区”
* **定位**: `patent_md_to_docx.py` -> `build_patent_docx()` 主函数末尾
* **修改要求**: 
  1. 删掉 Task 4 中手动添加的 `doc.add_page_break()`。
  2. 在追加完“说明书附图”标题后，**必须**再追加一个无边框、无特殊字体的干净空段落。这个段落作为“着陆区”，供用户安全地粘贴附图，阻断 H1 的边框向下级联。
* **代码指引**:
  ```python
  def build_patent_docx(md_text: str) -> Document:
      # ... [主循环] ...
      
      if not has_drawing_title:
          # 【核心修正 2】：删除了原来的 doc.add_page_break()，直接调用
          add_section_title(doc, "说明书附图")

      p_landing = doc.add_paragraph()
      p_landing.style = doc.styles['Normal']
      
      return doc
  ```
