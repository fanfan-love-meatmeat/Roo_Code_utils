# Task 4: H1 视觉边框重构与说明书附图自动挂载

> **执行人**: Python 开发团队
> **目标模块**: `tools/patent_style_utils.py` & `tools/patent_md_to_docx.py`
> **背景**: 需进一步逼近专利局官方排版视觉：仅保留加粗的下边框，强行确保 3 个空格字距，并在文档末尾物理兜底生成“说明书附图”标题。

## Action Item 1: H1 标题的底层重构 (单下边框加粗 + 暴力字距)
* **定位**: `patent_style_utils.py` -> `add_section_title()`
* **修改要求**:
  1. **移除上边框**：删除 `<w:top>` 的注入逻辑。
  2. **加粗下边框**：将 `w:sz` 的值从 `4` (0.5pt) 修改为 `12` (1.5pt) 或 `18` (2.25pt)。
  3. **暴力字距处理**：过滤原有字符串中的所有空白字符，严格遍历强制拼接。
* **重构代码指引**:
  ```python
  import re
  from docx.oxml.shared import OxmlElement
  from docx.oxml.ns import qn
  from docx.shared import Pt

  def add_section_title(doc, text: str):
      p = doc.add_paragraph()
      p.alignment = WD_ALIGN_PARAGRAPH.CENTER
      p.paragraph_format.space_before = Pt(12)
      p.paragraph_format.space_after = Pt(12)
      
      # 1. 暴力清洗与严格 3 空格注入
      # 抹除 (图1) 或 （图1）
      clean_text = re.sub(r'[\(（]图\d+[\)）]', '', text).strip()
      # 抹除所有全角/半角空格
      clean_text = re.sub(r'\s+', '', clean_text)
      # 强行用 3 个半角空格连接每一个中文字符
      spaced_text = '   '.join(list(clean_text))
      
      run = p.add_run(spaced_text)
      set_run_font(run, '楷体_GB2312', 'Times New Roman', Pt(18), bold=False)
      
      # 2. OOXML 注入：仅保留加粗下边框
      pPr = p._element.get_or_add_pPr()
      pBdr = OxmlElement('w:pBdr')
      
      bottom = OxmlElement('w:bottom')
      bottom.set(qn('w:val'), 'single')
      bottom.set(qn('w:sz'), '18')  # 18 = 2.25pt, 线条显著加粗
      bottom.set(qn('w:space'), '1')
      bottom.set(qn('w:color'), 'auto')
      
      pBdr.append(bottom)
      pPr.append(pBdr)
      
      return p
  ```

## Action Item 2: 末尾自动挂载《说明书附图》
* **定位**: `patent_md_to_docx.py` -> `build_patent_docx()` 主函数末尾。
* **修改要求**: 大模型生成的 Markdown 通常不包含最后的附图占位标题。为实现全自动排版闭环，需在解析完所有的 AST Token 后，于返回 `doc` 对象之前，强行调用 `add_section_title` 追加该页面。
* **代码指引**:
  ```python
  def build_patent_docx(md_text: str) -> Document:
      # ... [现有的 Markdown Token 遍历主循环] ...
      
      # 在循环结束，return doc 之前进行物理兜底：
      # 强行追加分页符和“说明书附图”标题
      doc.add_page_break()
      add_section_title(doc, "说明书附图")
      
      return doc
  ```

## 验收标准 (Acceptance Criteria)
1. 生成的 DOCX 中，所有一级大标题（如“权利 要求 书”）仅有底部边框，且线条明显变粗。
2. 任何一级大标题的文字之间绝对呈现 3 个空格的宽度，无粘连。
3. 无论输入的 Markdown 是否包含，生成的 Word 文档最后一页必然是带有下边框的 `# 说   明   书   附   图` 大标题。
