# Task 6: MathType 宏转换的 TeX 语法级清洗与幂等性防御

> **执行人**: Python 开发团队
> **目标模块**: `tools/patent_style_utils.py`
> **背景**: 当前的裸 LaTeX 向下取整/向上取整符号 (`\lfloor`, `\lceil`) 在触发 MathType 宏转换时，会因中文字体上下文冲突导致渲染崩溃（出现乱码）。需在 Python 写入 Word 之前，强制对这些定界符进行 `\left` 和 `\right` 结构化升格。为防止大模型偶然输出正确的结构导致重复前缀（如 `\left\left`），必须采用“先剥离、后穿衣”的幂等性清洗策略。

## Action Item 1: 文本 Run 写入前的幂等性清洗 (Inline Formula)
* **定位**: `patent_style_utils.py` -> `add_rich_paragraph()` 函数。
* **修改要求**: 在处理 `child.type == 'text'` 且准备调用 `p.add_run(txt)` 之前，拦截 `txt` 变量，执行严格的降级与安全包装逻辑。
* **代码指引**:
  ```python
  # ... [前序 AST 解析逻辑] ...
  
  txt = child.content
  if not txt.strip():
      continue

  # 【核心修正 1】：MathType 语法级幂等性防御 (向下取整)
  txt = txt.replace(r'\left\lfloor', r'\lfloor').replace(r'\right\rfloor', r'\rfloor')
  txt = txt.replace(r'\lfloor', r'\left\lfloor').replace(r'\rfloor', r'\right\rfloor')
  
  # 【核心修正 2】：MathType 语法级幂等性防御 (向上取整)
  txt = txt.replace(r'\left\lceil', r'\lceil').replace(r'\right\rceil', r'\rceil')
  txt = txt.replace(r'\lceil', r'\left\lceil').replace(r'\rceil', r'\right\rceil')

  # 原样单 Run 写入，保证宏连贯性
  run = p.add_run(txt)
  run.bold = in_strong
  # ... [后续字体设置] ...
  ```
