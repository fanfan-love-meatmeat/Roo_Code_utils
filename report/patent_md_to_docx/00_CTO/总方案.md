# TASK: 专利基础排版映射规范制订 (Patent MD-to-DOCX Style Mapping)

## 1. 任务背景 (Context)
公式逆向转换自动化管线（LaTeX -> OMML）暂时挂起。当前冲刺优先级变更为：确保专利纯文本、段落、各级标题等基础结构，能够100%无损、精准地映射到符合 CNIPA（国家知识产权局）审查规范的 Word 格式中。

## 2. 目标 (Objective)
输出一份名为 `映射表格.md` 的系统级规范文档。该文档必须穷尽所有非公式类的 Markdown AST 元素，并严格对齐到 `python-docx` 的底层操作 API 和 Word 属性字典。此表格将作为后续重构 `patent_md_to_docx.py` 核心解析器的唯一开发基准（Source of Truth）。

## 3. 范围与约束 (Scope & Constraints)
* **涵盖对象**：标题 (H1-H6)、标准段落、带自动编号段落 (`[NNNN]`)、行内样式 (粗体 `strong`、斜体 `em`)、图片引用 (`image`)。
* **排除对象**：LaTeX 公式（内联 `$`, 块级 `$$`）、嵌套表格。
* **技术底座**：映射起点必须是 `markdown-it-py` 解析出的 AST Token 语法，终点必须是 `python-docx` 的对象操作。
* **合规红线**：中文字体强制底层绑定“宋体” (SimSun)，英文字体强制绑定“Times New Roman”，首行缩进、行距需严格符合专利局规范。

## 4. 交付物要求 (Deliverable Specifications)
`映射表格.md` 必须至少包含以下维度的结构化数据。请团队按照下方的表格骨架进行全量分析与填充：

| Markdown 元素 / AST Token | 专利排版语义 | `python-docx` 实现路径 / 核心 API | 字体与段落属性 (格式约束) | 异常与边界情况防御 |
| :--- | :--- | :--- | :--- | :--- |
| `heading` (Level 1) | 发明名称 / 一级模块 | `doc.add_heading(text, level=1)` | 黑体，三号，居中，段前段后 0.5 行 | 需清除内部的 `strong` 标记，避免底层重复渲染加粗 |
| `paragraph` (带 `[0001]`) | 说明书正文段落 | `p = doc.add_paragraph()` | 中文宋体/英文TNR，小四，首行缩进 2 字符 | 需利用正则安全提取 `[0001]` 前缀，并将其作为独立 `Run` 进行加粗 |
| `paragraph` (无编号) | 权利要求 / 普通文本 | `p = doc.add_paragraph()` | 中文宋体/英文TNR，小四，悬挂缩进 | 需与带编号段落做好逻辑隔离与上下文切分 |
| `em` / `strong` | 强调内容 | `run.italic = True` / `run.bold = True` | 强制继承当前父级段落的字体族设置 | 跨段落或嵌套标签时的解析穿透问题规避 |
| `image` | 附图插入 | `doc.add_picture(path, width=...)` | 居中对齐，图片下方需自动生成图题占位符 | 相对路径断裂或文件缺失时的降级占位机制 |

## 5. 验收标准 (Acceptance Criteria)
1. 映射表必须 100% 覆盖 `raw_md` 产物中出现的所有非公式排版元素。
2. `python-docx` 的实现路径必须精确到 XML 属性级别（例如：明确指出如何通过 `w:eastAsia` 命名空间强绑定中文字体）。
3. 杜绝任何模棱两可的伪代码，所有映射逻辑必须在技术上已被验证可行。
