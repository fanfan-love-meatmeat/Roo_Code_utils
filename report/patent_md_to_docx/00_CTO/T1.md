# Sprint 1 研发任务书：底层原子操作层 (DOCX Builder)

## 0. 核心研发纪律 (The Golden Rules)
1. **绝对隔离**：本 Sprint 内的**任何**代码文件，绝对不允许出现 `import markdown_it` 或任何与 Markdown 解析相关的逻辑。本层只做纯粹的 DOCX 构建。
2. **测试驱动开发 (TDD)**：每一个原子函数必须配有对应的**属性断言测试 (Property Assertion Test)**。测试方法必须是：调用函数生成 DOCX -> 内存保存或落盘 -> 重新读取底层 XML 节点 -> 断言属性值。
3. **接口契约**：所有传入的 `text` 参数必须是纯文本，不包含任何 Markdown 标记（如 `**` 或 `#`）。

## Task 1.1: 专利文档初始化引擎
**描述**：构建全局 `Document` 对象，并强行注入符合 CNIPA 标准的物理页面属性。
* **函数签名**: 
  `def init_patent_document() -> Document:`
* **规格要求**:
  * 纸张大小：A4 (宽 21cm, 高 29.7cm)
  * 页边距：上 25mm, 下 15mm, 左 25mm, 右 15mm
* **【可验证性 / 测试断言】**:
  * 断言 `doc.sections[0].page_width.cm` 约等于 `21.0`
  * 断言 `doc.sections[0].top_margin.cm` 约等于 `2.5`
  * 断言 `doc.sections[0].bottom_margin.cm` 约等于 `1.5`

## Task 1.2: 字体 XML 注入引擎
**描述**：解决跨平台字体回退问题，实现绝对可靠的中英双语字体与字号控制。
* **函数签名**: 
  `def set_run_font(run: Run, cn_font: str = '宋体', en_font: str = 'Times New Roman', size: Pt = Pt(12), bold: bool = False) -> None:`
* **规格要求**:
  * 必须使用 `lxml` 底层操作，注入 `w:eastAsia`, `w:ascii`, `w:hAnsi`。
  * **必须注入 `w:hint="eastAsia"` 防御标签**。
* **【可验证性 / 测试断言】**:
  * 创建一个空段落，添加 run 并调用此函数。
  * 提取 `run._element.rPr` 的 XML 字符串。
  * 断言 XML 树中存在 `<w:rFonts>` 标签，并且其属性包含 `w:eastAsia="宋体"`, `w:ascii="Times New Roman"`, `w:hint="eastAsia"`。
  * 断言 `run.font.size == Pt(12)` 且 `run.font.bold == bold`。

## Task 1.3: 专利结构化段落构建器
**描述**：封装专利正文中出现的三种核心段落形态。
* **函数集合**:
  1. `def add_section_title(doc: Document, text: str) -> Paragraph:`
     * *规格*：黑体 (需调用 1.2 注入)，18pt (小二)，居中对齐，段前 Pt(12)，段后 Pt(6)。
  2. `def add_body_paragraph(doc: Document, text: str, number_prefix: str = None) -> Paragraph:`
     * *规格*：宋体+TNR，12pt，**首行缩进 Pt(24)**，1.5倍行距。
     * *逻辑*：如果传入了 `number_prefix` (如 `"[0001]"`), 必须将其作为一个独立的 `Run` 插入到最前面，并设置为**加粗**。正文 `text` 不加粗。
  3. `def add_claim_paragraph(doc: Document, claim_number: str, text: str) -> Paragraph:`
     * *规格*：宋体+TNR，12pt，**无首行缩进**，1.5倍行距。`claim_number` (如 `"1. "`) 需加粗。
* **【可验证性 / 测试断言】**:
  * 生成这三种段落后，通过 `p.paragraph_format.first_line_indent` 断言首行缩进值（精确匹配 `Pt(24)` 或 `None`）。
  * 断言 `p.alignment` 是否符合预期（居中/左对齐）。
  * 针对有编号的段落，断言 `p.runs[0].text` 是编号，且 `p.runs[0].bold == True`，而 `p.runs[1].bold == False/None`。

## Task 1.4: 视觉占位与降级构建器
**描述**：处理当前版本暂不进行底层转换的公式与图片，提供人工审阅友好的视觉占位符。
* **函数集合**:
  1. `def add_formula_placeholder(doc: Document, latex_text: str, is_inline: bool = True) -> None:`
     * *规格 (is_inline=True)*：不新建段落，直接在当前最后一个段落末尾添加深红色 (RGB: 192, 0, 0) 的 `Run`，内容为 `$latex_text$`。
     * *规格 (is_inline=False)*：新建独立段落，居中，设置段落底层 XML 底纹 `<w:shd w:fill="F2F2F2" w:val="clear"/>` (浅灰色)，深红文字，内容为 `【公式】 $$latex_text$$`。
  2. `def add_image_or_placeholder(doc: Document, img_abs_path: str, caption: str = "") -> None:`
     * *规格*：如果路径存在，直接 `add_picture` (宽14cm)，居中。
     * *降级规格*：如果路径不存在，使用 `PIL` 生成一个 400x200 的灰色占位图，并写入文字 `[图片缺失] 文件名`。将其保存为临时文件并插入文档，居中。最后附加图题段落。
* **【可验证性 / 测试断言】**:
  * 块级公式：解析生成的 paragraph 的 `pPr` XML 树，断言存在 `<w:shd w:fill="F2F2F2"/>`。断言 `run.font.color.rgb == RGBColor(192, 0, 0)`。
  * 图片降级：传入一个故意写错的路径，断言文档中是否成功插入了 `InlineShape`，并断言占位图临时文件被成功创建且未抛出崩溃异常。
