# Task 3: 排版引擎深度视觉复刻与样式微调

> **执行人**: Python 开发团队
> **目标模块**: `tools/patent_style_utils.py` & `tools/patent_md_to_docx.py`
> **背景**: 根据最新的人类专家视觉审查结果，当前的 DOCX 引擎在 H1 的边框渲染、H2/H3 的字体字号（宋体小四）以及局部对齐特例上存在偏差。本迭代需直接操作 OOXML 底层完成视觉级 1:1 复刻。

## Action Item 1: H1 一级标题终极形态 (上下边框 + 3空格字距 + 去图号)
* **定位**: `patent_style_utils.py` -> `add_section_title()`
* **问题**: 之前使用的是 `run.underline = True`，导致下划线紧贴文字且不贯穿页面。需要改为“段落边框”。
* **修改要求**:
  1. **清洗与字距重构**：用正则抹除标题中可能存在的 `（图X）` 或 `(图X)`。剔除所有原有空格后，强制在每个中文字符之间插入 **3 个空格**（例如：`说   明   书   摘   要`）。
  2. **删除旧下划线**：移除 `run.underline = True`。
  3. **注入段落边框 (OOXML)**：在段落级别插入 `<w:pBdr>` 标签，绘制上下边框。
* **参考实现代码**:
```python
  import re
  from docx.oxml.shared import OxmlElement
  from docx.oxml.ns import qn
  from docx.shared import Pt

  def add_section_title(doc, text: str):
      p = doc.add_paragraph()
      p.alignment = WD_ALIGN_PARAGRAPH.CENTER
      p.paragraph_format.space_before = Pt(12)
      p.paragraph_format.space_after = Pt(12)
      
      # 1. 文本预处理：去图号，加3个空格
      clean_text = re.sub(r'[\(（]图\d+[\)）]', '', text).strip()
      clean_text = clean_text.replace(' ', '')
      spaced_text = '   '.join(list(clean_text))
      
      run = p.add_run(spaced_text)
      set_run_font(run, '楷体_GB2312', 'Times New Roman', Pt(18), bold=False)
      
      # 2. OOXML 注入：段落上下边框
      pPr = p._element.get_or_add_pPr()
      pBdr = OxmlElement('w:pBdr')
      
      top = OxmlElement('w:top')
      top.set(qn('w:val'), 'single')
      top.set(qn('w:sz'), '4')  # 边框粗细
      top.set(qn('w:space'), '1')
      top.set(qn('w:color'), 'auto')
      
      bottom = OxmlElement('w:bottom')
      bottom.set(qn('w:val'), 'single')
      bottom.set(qn('w:sz'), '4')
      bottom.set(qn('w:space'), '1')
      bottom.set(qn('w:color'), 'auto')
      
      pBdr.append(top)
      pBdr.append(bottom)
      pPr.append(pBdr)
      
      return p
  ```

## Action Item 2: H2 二级标题的降维与特例分化
* **定位**: `patent_style_utils.py` -> `add_sub_title()`
* **问题**: 之前使用了 14pt (四号) 的黑体，实际上官方模板使用的是 12pt (小四) 的宋体加粗。且“摘要附图”需要单独居中。
* **修改要求**:
  1. **字号与字体修正**：将字体强制设为 `'宋体'`，字号设为 `Pt(12)`，保持加粗 `bold=True`。
  2. **对齐方式分化**：
     - 如果传入的 `text` 包含 `"摘要附图"`，则 `p.alignment = WD_ALIGN_PARAGRAPH.CENTER`。
     - 否则（如“技术领域”、“背景技术”等），设置 `p.alignment = WD_ALIGN_PARAGRAPH.LEFT` 且 `p.paragraph_format.first_line_indent = 0`（绝对左侧顶格）。

## Action Item 3: H3 三级标题的字号对齐
* **定位**: `patent_md_to_docx.py` 主循环中处理 `level == 3` 的分支（例如“### 实施例 1：...”）。
* **修改要求**:
  1. 将原本可能设定的 14pt 字体，统一下调为 **`Pt(12)` (小四号)**。
  2. 保持 `'宋体'`，加粗 `bold=True`，左对齐顶格（无缩进）。

## 验收标准 (Acceptance Criteria)
1. 生成的 DOCX 中，“权利要求书”、“说明书摘要”等一级标题，文字间必须有 3 个空格宽度的字距，且段落的上方和下方出现横跨页面宽度的细实线边框。不会出现“（图1）”字样。
2. “技术领域”、“发明内容”等二级标题为宋体、小四号、加粗、靠左顶格。
3. “摘要附图”二级标题为宋体、小四号、加粗、居中。
4. “实施例 1：...” 等三级标题为宋体、小四号、加粗。
