# 风险评估 — patent_md_to_docx CLI 工具固化

> **审计时间**: 2026-06-01 01:07 CST  
> **审计版本**: v1.0  
> **关联报告**: S2 项目现状梳理_20260601_0106_v1.0.md  
> **审查范围**: `tools/patent_md_to_docx.py` 追加 `__main__` CLI 块

---

## 1. 风险排序

### [P1-高危] `python -m` 模块调用在任意目录不可达

**风险**：S2 报告中设计的 `python -m tools.patent_md_to_docx` 命令**仅在项目根目录或 PYTHONPATH 已配置时有效**。用户从 `D:\OtherProject\` 执行该命令会报 `ModuleNotFoundError: No module named 'tools'`。

**原因**：`-m` 依赖 Python 的模块搜索路径，不会自动从脚本路径推断项目根。

**止血方案**：CLI 入口同时支持两种调用方式：

```python
if __name__ == '__main__':
    # 自动将项目根注入 sys.path，确保内部导入可用
    import sys, os
    sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
    # ...
```

然后 CLI 应被设计为**脚本执行**模式，而非模块模式：

```bash
# 正确用法（脚本执行，自动处理 sys.path）
python D:\path\to\tools\patent_md_to_docx.py input.md

# 错误用法（-m 模块调用，需额外配置）
python -m tools.patent_md_to_docx input.md
```

**建议**：将 CLI 入口放在项目根下的独立脚本 `patent_cli.py`，而非嵌入 `tools/patent_md_to_docx.py`。这样 `python patent_cli.py input.md` 在任何项目内都可工作，且职责分离更清晰。

---

### [P1-高危] 内部模块导入链需手动修正

**风险**：`build_patent_docx()` 导入链：

```python
# patent_md_to_docx.py 顶部
from tools.patent_style_utils import (
    init_patent_document, add_section_title, ...
)
```

当 `tools/patent_md_to_docx.py` 作为 CLI 入口直接运行时，`__name__` 从 `tools.patent_md_to_docx` 变为 `__main__`，Python 的导入解析器可能无法正确解析 `from tools.patent_style_utils import ...` 的包内导入。

**止血方案**：CLI 入口必须在任何 `from tools.xxx import` 之前注入 `sys.path`。S2 骨架中已包含此行，但需**放在文件最顶部**（在所有导入之前），而非 `if __name__ == '__main__':` 块内部。

---

### [P2-隐患] 中文路径在 PowerShell CLI 参数中的编码问题

**风险**：Windows PowerShell 5.1 中，包含中文的文件路径作为 CLI 参数传递时，可能因编码页 (cp936) 与 Python UTF-8 之间的转换导致 `FileNotFoundError`。

**触发条件**：
```powershell
python patent_cli.py "D:\test\patent_md\完整专利申请书_v2.4.md"
```

**止血方案**：CLI 入口在 `open()` 前显式尝试多种编码：

```python
try:
    with open(md_path, 'r', encoding='utf-8') as f: ...
except UnicodeDecodeError:
    import locale
    with open(md_path, 'r', encoding=locale.getpreferredencoding()) as f: ...
```

---

### [P2-隐患] 并发写入时间戳碰撞

**风险**：同一秒内两次 CLI 调用同一 MD 文件，`datetime.now().strftime('%Y%m%d_%H%M')` 生成相同时间戳，第二次调用写入覆盖第一次输出，无警告。

**止血方案**：写入前检测文件是否已存在，若存在则追加毫秒后缀或直接报错。

---

### [P2-隐患] CLI 入口与 `tools/` 目录语义冲突

**风险**：`tools/` 目录下的文件目前全部是**纯模块**（无可执行入口）。在其中混入一个带 `__main__` 的文件打破了目录语义一致性。未来其他 `tools/` 模块也可能需要 CLI 入口，导致 `tools/` 目录职责膨胀。

**止血方案**：将 CLI 入口独立为项目根目录下的 `patent_cli.py`：

```
Roo_Code_utils/
├── patent_cli.py          ← CLI 入口（~35 行）
├── server.py              ← MCP 入口
└── tools/
    ├── patent_md_to_docx.py   ← 纯引擎（无 __main__）
    └── patent_style_utils.py  ← 纯工具
```

## 2. 修正后的 CLI 方案

### 2.1 推荐架构

```python
# patent_cli.py （项目根目录下，35 行）

"""专利 MD → CNIPA 合规 DOCX CLI 转换器。

用法:
    python patent_cli.py path/to/input.md
    python patent_cli.py path/to/input.md -o path/to/output.docx
"""

import sys, os
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))

import argparse, datetime
from tools.patent_md_to_docx import build_patent_docx

parser = argparse.ArgumentParser(description='专利 MD → DOCX 转换器')
parser.add_argument('md_path', help='Markdown 文件路径')
parser.add_argument('-o', '--output', help='输出 DOCX 路径（可选）')
parser.add_argument('--no-timestamp', action='store_true', help='禁用时间戳')
args = parser.parse_args()

md_path = os.path.abspath(args.md_path)
if not os.path.isfile(md_path):
    parser.error(f'文件不存在: {md_path}')

with open(md_path, 'r', encoding='utf-8') as f:
    doc = build_patent_docx(f.read())

if args.output:
    out = args.output
else:
    fn = os.path.splitext(os.path.basename(md_path))[0]
    ts = '' if args.no_timestamp else f'_{datetime.datetime.now().strftime("%Y%m%d_%H%M")}'
    out_dir = os.path.join(os.path.dirname(os.path.dirname(md_path)), 'patent_docx')
    os.makedirs(out_dir, exist_ok=True)
    out = os.path.join(out_dir, f'{fn}{ts}.docx')

doc.save(out)
print(out)
```

### 2.2 调用方式

```bash
# 从任何目录调用
python D:\...\Roo_Code_utils\patent_cli.py test\patent_md\xxx.md

# 或在项目根目录下
cd Roo_Code_utils
python patent_cli.py test\patent_md\xxx.md
```
