## 0. 新 MD 源头结构变更分析

### 0.1 v1.0 旧源 vs v2.1 新源核心差异

| 维度 | 旧 MD (v1.0) | 新 MD (v2.1) | 影响 |
|------|-------------|-------------|------|
| **标题语法** | 无 `#` heading，纯粗体 `**技术领域**` + 空格中文字符串 | **完整 Markdown heading 层级**: `#` 一级 / `##` 二级 / `###` 三级 | ✅ T1 原方案 `doc.add_heading()` 可直接使用，无需启发式分类器 |
| **段落编号** | `**[0001]** ** ** 正文...` (粗体编号+粗体空格) | `[0001] 正文...` (纯文本编号) | ✅ 简化：无 bold spaces 污染，正则提取更简单 |
| **章节分隔** | `---` 水平线分隔三大块 | heading 层级自然分隔，无 `---` | ✅ 无需处理 hr token |
| **子标题** | `**技术领域**` → `**背景技术**` → ... (纯粗体行) | `## 技术领域` → `## 背景技术` → ... (AST heading token) | ✅ `classify_section` 从 30 行正则简化为 2 行 |
| **实施例** | `**实施例 N**` (粗体行) | `### 实施例 N：xxx` (H3 heading) | ✅ AST heading token 直接识别 |
| **公式** | 不变：行内 `$...$` + 独立行块公式 | 不变 + 增加了无 `$$` 包裹的独立公式行 | ⚠️ 需区分"独立行公式"与"普通文本行" |
| **表格** | 不变：`| ... |` | 不变 | ✅ 映射方案不变 |
| **图片** | `![](.md/figures/*)` | 本次 MD 无图片引用 | — 保留映射能力 |

### 0.2 新 MD 的 markdown-it-py AST 实测结果

用 markdown-it-py 2.2.0 解析新 MD 后，**自检验证通过**的全量 Token 分布:

```
Token 类型              数量
──────────────────────────────
inline                   123
paragraph_open / close   110 each
list_item_open / close    17 each
heading_open / close      13 each
ordered_list_open / close  3 each
hr                        0
image                     0
table_open                1
```

关键发现:

```
# 说明书摘要        → heading_open(h1) → inline → heading_close    ✅ h1: 3 个
## 技术领域          → heading_open(h2) → inline → heading_close    ✅ h2: 6 个
### 实施例 1         → heading_open(h3) → inline → heading_close    ✅ h3: 4 个
[0001] 本发明...    → paragraph_open → inline → paragraph_close   ✅ 33 段编号段落
S1. 构建...         → paragraph_open → inline → paragraph_close   ✅ 4 个步骤子项
1. 一种...          → ordered_list_open → list_item_open → inline ⚠️ 17 项权利要求
| 符号 | 物理含义 |   → table_open → ... → table_close             ✅ 1 张表格
```

**⚠️ 关键修正**: 权利要求（`1. 一种基于空间侵占反馈...`）在 CommonMark 解析器中被识别为**有序列表** (`ordered_list`)，而非普通段落。这意味着由 3 个 `ordered_list` 块（分别包含 1/15/1 项）承载全部 17 项权利要求。主循环必须处理 `ordered_list_open` → `list_item_open` 的嵌套结构。**v2.0 映射表已据此修正。**

**结论**: T1 规范的原始假设——"映射起点是 markdown-it-py 的 AST Token"——在新 MD 源头下**完全成立**。不再需要 `classify_section()` 的复杂启发式正则。`heading_open` token 直接可用，`paragraph_open` 仅需区分 claim/numbered/normal 三种语义。

```python
def set_run_font(run, cn_font='宋体', en_font='Times New Roman',
                 size=Pt(12), bold=False):
    run.font.name = en_font
    run.font.size = size
    run.font.bold = bold
    rPr = run._element.get_or_add_rPr()
    rFonts = rPr.find(qn('w:rFonts'))

    rFonts.set(qn('w:ascii'), en_font)
    rFonts.set(qn('w:hAnsi'), en_font)
```

### 2.3 行内元素 → DOCX Run

| AST Token | 语义 | DOCX 实现 | 边界防御 |
|:---|:---|:---|:---|
| `strong_open → text(X) → strong_close` | 粗体 | `run.bold = True` + `set_run_font(run,...)` | `X.strip() == ''` → 跳过 |
| `em_open → text(X) → em_close` | 斜体 | `run.italic = True` | 极少使用，保留 |
| `text` (无修饰) | 正文 | `run = p.add_run(X)` + `set_run_font(run,...)` | 连续 text 合并为单 Run |
| 行内 `$latex$` (在 `text` content 中) | 行内公式占位 | 检测 inline.content 中的 `$...$` 片段 → 拆分 text token 为"前文 + 红字公式 + 后文"三个 Run | 红字 `RGBColor(0xC0,0x00,0x00)`; 不另起行; 识别 `$$` (误判)→降级为行内 |
| `softbreak` / `hardbreak` | 换行 | 忽略 / `run.add_break()` | 极少出现 |

## 3. 核心实现骨架 v2.0

### 3.1 初始化

```python
from docx import Document
from docx.shared import Cm, Pt, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.oxml.ns import qn
import markdown_it
import re

def init_patent_document() -> Document:
    doc = Document()
    style = doc.styles['Normal']
    style.font.name = 'Times New Roman'
    style.font.size = Pt(12)
    style.element.rPr.rFonts.set(qn('w:eastAsia'), '宋体')
    for sec in doc.sections:
        sec.top_margin = Cm(2.5)
        sec.bottom_margin = Cm(1.5)
        sec.left_margin = Cm(2.5)
        sec.right_margin = Cm(1.5)
    return doc
```

### 3.2 主循环（v2.0 简化版）

```python
def build_patent_docx(md_text: str) -> Document:
    md = markdown_it.MarkdownIt('commonmark', {'breaks': False})
    tokens = md.parse(md_text)
    doc = init_patent_document()
    i, prev_was_h1 = 0, False

    while i < len(tokens):
        t = tokens[i]

        if t.type == 'heading_open':
            level = int(t.tag[1])
            text = tokens[i + 1].content.strip()
            if level == 1:
                p = doc.add_paragraph(); p.alignment = CENTER
                run = p.add_run(text); set_run_font(run, size=Pt(18))
                prev_was_h1 = True
            elif level == 2:
                p = doc.add_paragraph(); p.alignment = CENTER
                run = p.add_run(text); run.bold = True; set_run_font(run, size=Pt(14))
            elif level == 3:
                p = doc.add_paragraph()
                run = p.add_run(text); run.bold = True; set_run_font(run, size=Pt(14))
            i += 3

        elif t.type == 'ordered_list_open':
            # 权利要求 — CommonMark 将 "1. xxx" 解析为有序列表
            claim_num = 1
            start_attr = t.attrGet('start')  # 支持非1起始编号
            if start_attr:
                claim_num = int(start_attr)
            i += 1
            while i < len(tokens) and tokens[i].type != 'ordered_list_close':
                if tokens[i].type == 'list_item_open':
                    # 收集 list_item 内所有 paragraph 的子 token
                    body_texts = []
                    i += 1
                    while i < len(tokens) and tokens[i].type != 'list_item_close':
                        if tokens[i].type == 'paragraph_open':
                            inline = tokens[i + 1]
                            if inline.content.strip():
                                body_texts.append(inline.content.strip())
                            i += 3
                        else:
                            i += 1
                    # 合并多段为一项权利要求
                    body = '; '.join(body_texts) if body_texts else ''
                    add_claim_paragraph(doc, claim_num, body)
                    claim_num += 1
                else:
                    i += 1
            i += 1  # 跳过 ordered_list_close

        elif t.type == 'paragraph_open':
            inline = tokens[i + 1]
            raw = inline.content.strip()

            if prev_was_h1 and raw:
                # 发明名称
                p = doc.add_paragraph(); p.alignment = CENTER
                run = p.add_run(raw); run.bold = True; set_run_font(run, size=Pt(16))
                doc.add_paragraph()  # 空一行
                prev_was_h1 = False
            elif re.match(r'^\[(\d{4})\]\s', raw):
                # 编号段落
                m = re.match(r'^\[(\d{4})\]\s(.*)', raw.replace('** ** ', ''))
                add_numbered_paragraph(doc, m.group(1), m.group(2))
            elif re.match(r'^S\d+\.\s', raw):
                # S步骤行
                m = re.match(r'^(S\d+\.)\s(.*)', raw)
                add_step_paragraph(doc, m.group(1), m.group(2))
            elif raw and _is_standalone_formula(raw):
                add_display_formula(doc, raw)
            elif raw:
                add_rich_paragraph(doc, inline)
            i += 3

        elif t.type == 'table_open':
            i = _parse_table(doc, tokens, i)

        else:
            i += 1

    return doc
```

### 3.3 行内元素处理

```python
def add_rich_paragraph(doc, inline_token):
    """解析 inline.children，生成带 Run 级样式的段落"""
    p = doc.add_paragraph()
    p.paragraph_format.first_line_indent = Pt(24)
    p.paragraph_format.line_spacing = 1.5
    children = inline_token.children if inline_token.children else []

    i, in_strong, in_em = 0, False, False
    while i < len(children):
        c = children[i]
        if c.type == 'strong_open':   in_strong = True; i += 1; continue
        if c.type == 'strong_close':  in_strong = False; i += 1; continue
        if c.type == 'em_open':       in_em = True; i += 1; continue
        if c.type == 'em_close':      in_em = False; i += 1; continue
        if c.type == 'text':
            txt = c.content
            if not txt.strip():
                i += 1; continue
            # 检测行内 $...$ 公式
            parts = re.split(r'(\$[^$]+\$)', txt)
            for part in parts:
                if part.startswith('$') and part.endswith('$'):
                    run = p.add_run(part)
                    run.font.color.rgb = RGBColor(0xC0, 0x00, 0x00)
                    set_run_font(run, size=Pt(12))
                elif part:
                    run = p.add_run(part)
                    run.bold = in_strong
                    run.italic = in_em
                    set_run_font(run, size=Pt(12))
        i += 1
    return p


def _is_standalone_formula(text: str) -> bool:
    """判断是否为独立公式行"""
    stripped = text.strip()
    return bool(re.match(r'^\$[^$]+\$$', stripped))  # 整行仅由 $...$ 组成
```

## 4. v1.0 → v2.0 核心变更清单

| 变更项 | v1.0 | v2.0 | 原因 |
|--------|------|------|------|
| **标题映射** | 启发式 `classify_section()` 正则分类器（30行） | **直接使用 `heading_open` token**（3行） | 新 MD 使用 `#` / `##` / `###` 语法 |
| **子标题映射** | `**技术领域**` 粗体行正则匹配 | `## 技术领域` → `heading_open(h2)` | 同上 |

| **权利要求映射** | `paragraph_open` + `N. ` 正则匹配 | `ordered_list_open` + `list_item_open` token 嵌套处理 | **自检发现**: CommonMark 解析器将 `1. xxx` 识别为有序列表，需消费 `list_item_open` 的嵌套 paragraph |
| **多段合并** | 未覆盖 | list_item 内多个 paragraph 合并：`'; '.join(texts)` | claim 1 内嵌 S1-S4 子步骤，需跨 paragraph 合并为单项权利要求 |
| **独立公式** | 仅 `$$...$$` 识别 | 扩展：`$...$` 包裹的独立行也识别为块公式 | 新 MD 中存在 `$V_overlap = N_collision · l³$` 独立行 |
| **章节分隔** | `---` → `doc.add_page_break()` | 无 `---`，heading 层级自然分隔 | 新 MD 结构变化 |
