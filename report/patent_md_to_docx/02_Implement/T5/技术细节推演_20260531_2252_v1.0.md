## 1. Action Item 1: H1 原生分页

### 1.1 技术原理

| 方案 | 机制 | 用户体验 |
|------|------|---------|
| **旧**: `doc.add_page_break()` | 插入物理分页符字符 `<w:br w:type="page"/>` | 按 Backspace 会删除分页符 → 标题倒吸至上一页 → 格式混乱 |
| **新**: `page_break_before = True` | 段落属性 `<w:pageBreakBefore/>` in `<w:pPr>` | 段落"永远"在新页开始。无物理字符，Backspace 无效。这是 Word 原生分页语义 |

### 1.2 应应用的范围

所有通过 `add_section_title()` 生成的 H1 标题自动获得该属性，包括：
- `# 说明书摘要`
- `# 权利要求书`
- `# 说明书`
- `# 说明书附图`（自动挂载）

首页 `# 说明书摘要` 虽设 `page_break_before = True`，但 Word 自动忽略首段，不会产生空白首页。

### 1.3 修改清单

| 文件 | 函数 | 修改 |
|------|------|------|
| `patent_style_utils.py` | `add_section_title()` | 新增 `p.paragraph_format.page_break_before = True` (+1 行) |

## 2. Action Item 2: 移除硬分页 + 追加着陆区

### 2.1 当前代码（T4 遗留）

```python
if not has_drawing_title:
    doc.add_page_break()                         # ← 删除此行
    add_section_title(doc, "说明书附图")
```

### 2.2 修改后

```python
if not has_drawing_title:
    add_section_title(doc, "说明书附图")          # H1 自带 page_break_before，自动分页

# 纯净着陆区：阻断 H1 边框和 18pt 字体向下级联
p_landing = doc.add_paragraph()
p_landing.style = doc.styles['Normal']
```

**着陆区作用**：
- 用户在 DOCX 末尾粘贴图片时，光标落在这个空段落中
- 因它是 Normal 样式（宋体 12pt, 无边框, 无缩进），粘贴的图片不会继承 H1 的边框格式或 18pt 字号
- 若没有这个着陆区，用户在 H1 段落后直接按 Enter 会继承 H1 的格式

| 风险项 | 严重度 | 说明 |
