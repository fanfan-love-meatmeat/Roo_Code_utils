## 2. 公式占位方案（T1 扩展需求）

### 2.1 需求定义

> "公式的话，只需要将 md 中的 latex 在 word 中进行占位即可。如果是一段话中的好几个变量符号，则无需另起一行；如果是独立的一个公式，则需另起一行。"

### 2.2 判定规则

```
LaTeX 在 MD 中的出现形式:
  ┌─ $...$ (行内公式, 单$) → Inline formula → 不另起行，与正文同段落
  ├─ $$...$$ (块级公式, 双$) → Display formula → 另起独立段落，居中
  └─ 独立行公式 (行内容完全为 $...$ 或 $$...$$) → 另起独立段落
```

### 2.3 映射方案

| MD 模式 | AST Token | DOCX 行为 | 实现 |
|---------|-----------|----------|------|
| 行内 `$x^2$` | `inline` → `text` (content 含 `$x^2$`) | 在当前段落 Run 中以红色高亮渲染公式原文 | `run.font.color.rgb = RGBColor(255,0,0)` + `run.font.highlight_color = WD_COLOR_INDEX.YELLOW` |
| 块级 `$$...$$` | `paragraph_open` → `inline` | 新段落，居中，红色高亮 + 灰色底纹 + 前缀"【公式占位】" | `p.alignment = CENTER` + `shd.fill = 'F2F2F2'` |

**设计原则**：
- **不丢失信息** — 公式原文以 LaTeX 形式保留在 DOCX 中，人工可读可编辑
- **可辨识** — 红色文字 + 灰色底纹明确标识"这是待转换的公式占位符"
- **保留行内/块级语义** — 行内公式在段落流中不打断文本，块级公式独立居中
- **无侵入** — 后续公式转换管线（LaTeX→OMML）可以直接搜索红色 Run 定位并替换为 OMML 公式节点

### 2.4 实现代码片段

```python
from docx.shared import RGBColor, Pt
from docx.oxml.ns import qn

def add_inline_formula_placeholder(paragraph, latex_text: str):
    """行内公式占位：红色文字，保留在段落流中"""
    run = paragraph.add_run(f"${latex_text}$")
    run.font.color.rgb = RGBColor(0xC0, 0x00, 0x00)  # 深红色
    run.font.size = Pt(12)
    set_run_font(run, '宋体', 'Times New Roman', Pt(12))

def add_display_formula_placeholder(doc, latex_text: str):
    """块级公式占位：独立段落，居中，灰色底纹"""
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    
    # 灰色底纹
    pPr = p._element.get_or_add_pPr()
    shd = pPr.makeelement(qn('w:shd'), {
        qn('w:fill'): 'F2F2F2',
        qn('w:val'): 'clear'
    })
    pPr.append(shd)
    
    # 公式文本
    run = p.add_run(f"【公式】 $${latex_text}$$")
    run.font.color.rgb = RGBColor(0xC0, 0x00, 0x00)
    run.font.size = Pt(12)
    set_run_font(run, '宋体', 'Times New Roman', Pt(12))
```

## 3. 图片占位方案（T1 扩展需求）

### 3.1 需求定义

> "图片只需要另起一行占位即可，目前还没有的。"

### 3.2 映射方案

| MD 模式 | AST Token | DOCX 行为 | 实现 |
|---------|-----------|----------|------|
| `![](.md/figures/xxx.png)` | `image` | 居中段落 + 灰色边框矩形 + 文件名标注 + "【图片待嵌入】"标记 | `p.alignment = CENTER` + Pillow 生成占位灰度图 |
| 图片文件可访问 | `image` + 文件存在 | 直接嵌入原图 + 居中对齐 | `doc.add_picture(path)` |
| 图片文件缺失 | `image` + 文件不存在 | 生成 200×100 灰色矩形占位图，标注文件名 | Pillow `Image.new('RGB', (400,200), 'E0E0E0')` |

**设计原则**：
- **渐进增强** — 有图则用原图，无图则用占位。后续补充图片后，直接覆盖占位段即可
- **可追溯** — 占位符包含文件名，方便后续人工定位和替换
- **视觉友好** — 灰色矩形占位 + 清晰标注，编辑者一眼可知此处需要图片

### 3.3 实现代码片段

```python
from docx.shared import Cm, Inches
from PIL import Image, ImageDraw, ImageFont
import os

def add_image_or_placeholder(doc, md_base_dir: str, img_rel_path: str, caption: str = ""):
    """嵌入图片或生成占位符"""
    img_abs_path = os.path.join(md_base_dir, img_rel_path)
    
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    
    if os.path.exists(img_abs_path):
        # 图片存在 — 直接嵌入
        run = p.add_run()
        run.add_picture(img_abs_path, width=Cm(14))
    else:
        # 图片缺失 — 生成占位图
        placeholder = Image.new('RGB', (400, 200), color=(0xE0, 0xE0, 0xE0))
        draw = ImageDraw.Draw(placeholder)
        draw.text((10, 80), f"[图片缺失]\n{os.path.basename(img_rel_path)}", 
                  fill=(0x80, 0x80, 0x80))
        temp_path = os.path.join(os.path.dirname(md_base_dir), f"_placeholder_{hash(img_rel_path)}.png")
        placeholder.save(temp_path)
        run = p.add_run()
        run.add_picture(temp_path, width=Cm(10))
    
    # 图题
    if caption:
        cap_p = doc.add_paragraph()
        cap_p.alignment = WD_ALIGN_PARAGRAPH.CENTER
        cap_run = cap_p.add_run(caption)
        cap_run.font.size = Pt(10)
        set_run_font(cap_run, '宋体', 'Times New Roman', Pt(10))
```

|---------|------|----------|------|
| `**text**` | 粗体 | `run.bold = True` | 若 text 为纯空白 → 跳过 |

### 4.4 字体设置的完整 XML 级代码（v2.0 修订版）

```python
from docx.oxml.ns import qn
from docx.shared import Pt

def set_run_font(run, cn_font='宋体', en_font='Times New Roman', 
                 size=Pt(12), bold=False):
    """
    一步到位设置中英文字体 + 字号 + 加粗。
    v2.0 新增: w:hint="eastAsia" 防御性渲染提示
    """
    run.font.name = en_font
    run.font.size = size
    run.font.bold = bold

    rPr = run._element.get_or_add_rPr()
    rFonts = rPr.find(qn('w:rFonts'))
    if rFonts is None:
        rFonts = rPr.makeelement(qn('w:rFonts'), {})
        rPr.insert(0, rFonts)

    rFonts.set(qn('w:eastAsia'), cn_font)   # 东亚字体: 宋体
    rFonts.set(qn('w:ascii'), en_font)       # ASCII字体: Times New Roman
    rFonts.set(qn('w:hAnsi'), en_font)       # 高ANSI字体: Times New Roman
    rFonts.set(qn('w:hint'), 'eastAsia')     # v2.0 新增: 强制渲染提示，阻断等线回退
```

## 5. 更新后的解析器架构（v2.0）

```
专利 MD 文本
    │
    ▼
┌─────────────────────────────────────┐
│  Phase 1: markdown-it-py Tokenization │
│  md.parse(md_text) → block_tokens    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Phase 2: 语义分类 (Heuristic Tagger)│
│  遍历 block_tokens:                  │
│    paragraph + classify_section()    │
│    inline → 检测 $...$ 公式 或 **    │
│    hr → PAGE_BREAK                   │
│    image → IMAGE_PLACEHOLDER         │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Phase 3: DOCX 构建 (Builder)        │
│  根据语义标签调用:                   │
│    SECTION_TITLE → centered 18pt     │
│    SUB_TITLE    → centered 14pt bold │
│    NUMBERED_PARA → indented Pt(24)   │
│    CLAIM        → no-indent + bold N │
│    DISPLAY_FORMULA → red + gray bg   │
│    INLINE_FORMULA → red inline run   │
│    IMAGE        → embed or placeholder│
│  v2.0: 全量使用修正后的页边距/缩进   │
└─────────────────────────────────────┘
    │
    ▼
  最终 DOCX 文件 (CNIPA 合规)
```
