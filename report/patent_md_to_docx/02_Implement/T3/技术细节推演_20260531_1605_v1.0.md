## 1. Action Item 1: H1 一级标题终极形态

### 1.1 技术原理

**`run.underline` vs `w:pBdr`**:

| 特性 | `run.underline = True` (当前) | `w:pBdr` top+bottom (T3 目标) |
|------|------|------|
| OOXML 层级 | Run 级 (`<w:u w:val="single"/>`) | 段落级 (`<w:pBdr><w:top/> <w:bottom/></w:pBdr>`) |
| 线条覆盖范围 | 仅覆盖文字宽度 | 横跨页面完整宽度 |
| 视觉效果 | 文字底部的细线 | 段落顶/底的贯穿线，更正式 |
| 符合专利模板 | ❌ 参考模板使用段落级分隔 | ✅ 更接近标准模板 |

**3空格字距**: 将 `说明书摘要` 变为 `说   明   书   摘   要`。这是中国公文和专利中常见的"字间加空"标题格式。

**去图号**: MD 中可能包含 `# 说  明  书  摘  要 （图1）`，需将 `（图1）` 去除后再排版。

### 1.2 边界属性说明

| 属性 | 值 | 含义 |
|------|-----|------|
| `w:val='single'` | 单实线 | 标准实线边框 |
| `w:sz='4'` | 4/8 pt = 0.5pt | 细线（Word 默认表格边框粗细） |
| `w:space='1'` | 1pt | 文字与边框间距 |
| `w:color='auto'` | 自动 | 使用段落字体颜色 |

### 1.3 修改清单

| 文件 | 函数 | 修改 |
|------|------|------|
| `patent_style_utils.py` | `add_section_title()` | 重写：去图号 → 3空格 → 段落边框 → 移除 underline |
| `patent_style_utils.py` | import | 新增 `from docx.oxml.shared import OxmlElement` + `import re` |

## 2. Action Item 2: H2 二级标题降维与特例分化

### 2.1 字体字号修正评估

**从 楷体_GB2312 Pt(14) 降到 宋体 Pt(12)**:

| 维度 | 旧 (T2+hotfix) | 新 (T3) | CNIPA 合规 |
|------|:---:|:---:|:---:|
| 中文字体 | 楷体_GB2312 | **宋体** | ✅ 在白名单 |
| 字号 | Pt(14) (四号) | **Pt(12) (小四)** | ✅ ≥小四 |
| 加粗 | True | True | — |
| 对齐 | LEFT | LEFT (摘要附图→CENTER) | — |

**评估**: 12pt 宋体加粗是 CNIPA 专利说明书中最标准的子标题格式。14pt 楷体是前期 T2 误用的结果。此次修正使 H2 回归标准。

### 2.2 对齐方式分化

```python
if '摘要附图' in text:
    p.alignment = CENTER
else:
    p.alignment = LEFT
    p.paragraph_format.first_line_indent = 0
```

**合理性**: 摘要附图在专利"五书"结构中与说明书摘要并列，属于特殊的一级附属节，居中排版符合视觉惯例。其他 H2（技术领域/背景技术/发明内容/附图说明/具体实施方式）均为说明书正文内的子章节，左对齐顶格更符合正文层级逻辑。

## 3. Action Item 3: H3 三级标题字号对齐

### 3.1 修改

| 文件 | 函数 | 修改 |

| `patent_style_utils.py` | `add_embodiment_title()` | 字号 Pt(14) → Pt(12) |

### 3.2 修改后代码

```python
def add_embodiment_title(doc, text):
    """实施例标题。宋体 12pt 加粗左对齐顶格。"""
    p = doc.add_paragraph()
    p.paragraph_format.first_line_indent = 0
    p.paragraph_format.space_before = Pt(12)
    p.paragraph_format.space_after = Pt(6)
    run = p.add_run(text)
    set_run_font(run, '宋体', 'Times New Roman', Pt(12), bold=True)
    return p
```

## 5. 风险评估

| 风险项 | 严重度 | 说明 |
|--------|:---:|------|
| `OxmlElement` 跨版本兼容 | P2 | 已在当前 python-docx 验证通过。备选: `pPr.makeelement(qn('w:pBdr'), {})` |
| 3空格修改文本本体 | P2 | 如果后续需要从 DOCX 提取标题文本，空格会被包含。不影响打印/渲染 |
