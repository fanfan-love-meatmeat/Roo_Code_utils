## 1. Action Item 1: H1 一级标题下划线与字体修正

### 1.1 下划线可行性

**OOXML 标准**: `run.underline = True` 生成 `<w:u w:val="single"/>`。

**python-docx API**: `run.underline = True` 等价于设置 `WD_UNDERLINE.SINGLE`。`True`/`False`/`None` 三态与 `bold`/`italic` 完全一致。

**CNIPA 规范**: 《专利审查指南》第一部分第一章要求"说明书第一页第一行应当写明发明名称，并左右居中"。对五大书标题（"权利要求书"、"说明书"等）的下划线**无明确要求或禁止**。属于**可自由发挥的排版选择**。

### 1.2 黑体 vs 宋体评估

| 来源 | 规定 | 黑体是否合规 |
|------|------|:---:|
| CNIPA 官网 (art_1517_92472) | "申请文件应当使用宋体、仿宋体或楷体" | ❌ **不在列** |
| 专利审查指南 | 未对标题字体做独立规定，默认继承正文规范 | ❌ 严格解读下不可 |
| 行业实践 (参考模板) | 标题使用宋体 (小二 18pt) | 行业主流 |
| T2 设计意图 | 视觉层级区分：H1 黑体 vs 正文 宋体 | 行业合理实践 |

**建议**: 黑体虽偏离 CNIPA 字面规范，但在内部 Draft 流转阶段是广泛接受的行业做法。若需绝对合规，可维持宋体。考虑到 T2 明确要求黑体，按需求执行。

### 1.3 修改清单

| 文件 | 函数 | 修改 |
|------|------|------|
| `patent_style_utils.py` | `add_section_title()` | `run.underline = True` + 字体改为黑体 |
| `patent_md_to_docx.py` | 主循环 `heading_open (h1)` | 无修改（已委托给 `add_section_title`） |

**修改后代码**:
```python
def add_section_title(doc: Document, text: str):
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    p.paragraph_format.space_before = Pt(12)
    p.paragraph_format.space_after = Pt(6)
    run = p.add_run(text)
    run.underline = True                          # T2: 强制单下划线
    set_run_font(run, '黑体', 'Times New Roman', Pt(18), bold=False)  # T2: 宋体→黑体
    return p
```

## 2. Action Item 2: H2 二级标题左对齐修正

### 2.1 对齐方式评估

当前 H2 为 CENTER 居中。T2 要求改为 LEFT 顶格。

**合理性**: 大量标准专利文档中，H2 级标题（如"技术领域"、"背景技术"）为左对齐加粗。居中样式通常保留给 H1 五大书标题。此修改**增强视觉层级区分**。

### 2.2 强制无缩进评估

H2 标题不应继承正文的首行缩进 (Pt(24))。当前 `add_sub_title()` 未显式设置 `first_line_indent`，继承自 `Normal` 样式（在 `init_patent_document` 中全局 `Normal` 样式也未设缩进）。但为防御性考虑，T2 要求显式声明 `first_line_indent = 0` 是合理的。

### 2.3 修改清单

| 文件 | 函数 | 修改 |
|------|------|------|
| `patent_style_utils.py` | `add_sub_title()` | `alignment = LEFT` + `first_line_indent = 0` + 黑体 |
| `patent_md_to_docx.py` | 主循环 `heading_open (h2)` | 无修改 |

**修改后代码**:
```python
def add_sub_title(doc: Document, text: str):
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.LEFT            # T2: CENTER → LEFT
    p.paragraph_format.first_line_indent = 0          # T2: 强制无缩进
    p.paragraph_format.space_before = Pt(12)
    p.paragraph_format.space_after = Pt(6)
    run = p.add_run(text)
    set_run_font(run, '黑体', 'Times New Roman', Pt(14), bold=True)  # T2: 宋体→黑体
    return p
```

## 3. Action Item 3: 数学公式绝对纯净度降维

### 3.1 技术原理：为什么单 Run 对 MathType 关键

**MathType "Toggle TeX" 机制** (来源: MathType 官方文档):

1. MathType 的 `Ctrl+\` 快捷键扫描当前光标位置的**连续文本**
2. 它识别 `$...$` 或 `$$...$$` 作为 TeX 方程边界
3. 将边界内的 TeX 代码转换为 MathType OLE 对象
4. **关键约束**: 如果 `$x^2$` 被 python-docx 拆分为多个 Run（如 `$` / `x^2` / `$` 三个独立 Run），MathType 的文本扫描在 Run 边界处断裂，无法识别完整的方程边界

**Word 内置方程编辑器**: `Alt+=` 打开方程编辑器后可输入 LaTeX。但它**不会自动扫描**文档中的 `$...$` 并批量转换——那是 MathType 的独有功能。

### 3.2 当前代码问题

**块级公式** (`add_display_formula_placeholder`):
```python
# 当前：灰色底纹 + 深红字体 + 前缀标记
pPr.append(shd)  # w:shd fill=F2F2F2
run.font.color.rgb = RGBColor(0xC0, 0x00, 0x00)
```
→ T2 要求：**全部删除**，改为纯黑 Times New Roman Pt(12) 居中

**行内公式** (`add_rich_paragraph`):
```python
# 当前：re.split 拆分 $...$ → 独立红字 Run
parts = re.split(r'(\$[^$]+\$)', txt)
for part in parts:
    if part.startswith('$') and part.endswith('$'):
        run = p.add_run(part)
        run.font.color.rgb = RGBColor(0xC0, 0x00, 0x00)  # 红色 Run
```
→ T2 要求：**彻底删除 split 逻辑**，直接 `p.add_run(txt)` 原样写入

### 3.3 修改清单

| 文件 | 函数/位置 | 修改 |
|------|----------|------|
| `patent_style_utils.py` | `add_display_formula_placeholder()` | 删除灰色底纹注入代码 + 删除红色字体；保留居中、段间距、纯黑 TNR Pt(12) |
| `patent_style_utils.py` | `add_rich_paragraph()` 内 `text` 处理 | 删除 `re.split(r'(\$[^$]+\$)', txt)` 和红色 Run 逻辑；改为 `p.add_run(txt)` 单 Run 原样写入 |

**修改后代码**:

```python
def add_display_formula_placeholder(doc: Document, latex_text: str):
    """块级公式占位。纯文本，居中，为 MathType 宏保留 $ 分隔符。"""
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    p.paragraph_format.space_before = Pt(6)
    p.paragraph_format.space_after = Pt(6)
    run = p.add_run(latex_text)  # 纯黑文本，单 Run，不含任何格式修饰
    set_run_font(run, '宋体', 'Times New Roman', Pt(12))
    return p


def add_rich_paragraph(doc: Document, inline_token, cn_font='宋体',
                       en_font='Times New Roman', size=Pt(12)):
    """处理包含行内元素的段落。T2: $...$ 公式不再拆分，作为普通文本单 Run 原样写入。"""
    p = _make_body_paragraph(doc)
    children = inline_token.children if inline_token.children else []

    in_strong = False
    in_em = False

    for child in children:
        if child.type == 'strong_open':
            in_strong = True
            continue
        if child.type == 'strong_close':
            in_strong = False
            continue
        if child.type == 'em_open':
            in_em = True
            continue
        if child.type == 'em_close':
            in_em = False
            continue
        if child.type != 'text':
            continue

        txt = child.content
        if not txt.strip():
            continue

        # T2: 直接原样写入，不拆分 $...$，保证 MathType 宏兼容
        run = p.add_run(txt)
        run.bold = in_strong
        run.italic = in_em
        set_run_font(run, cn_font, en_font, size)

    return p
```

## 5. 风险评估

| 风险项 | 严重度 | 说明 |
|--------|:---:|------|
| 黑体缺失降级 | P2 | 如果系统无黑体字体，Word 自动回退到默认中文字体（等线）。`w:hint='eastAsia'` 已设置，可部分缓解 |
| run.underline 无效果 | P2 | `run.underline = True` 在所有 python-docx 版本中已验证可用。仅在极旧版本 Word 中可能渲染异常 |
| MathType 版本差异 | P2 | `Ctrl+\` 仅在 MathType 7.x 中可用。MathType 6.x 使用不同快捷键。MathType 插件未安装时无影响（`$` 保留为普通文本） |
| 公式语义丢失 | P1 | 删除红字/灰底后，人工编辑者无法一眼区分公式与正文。但 T2 的设计意图是为 MathType 宏让路，视觉区分由宏完成 |
