## 1. Action Item 1: H1 单下边框 + 暴力字距重构

### 1.1 与 T3 代码对比

| 维度 | T3 (当前) | T4 (目标) |
|------|---------|---------|
| 边框 | `w:top` + `w:bottom`，`w:sz='4'`(0.5pt) | **仅 `w:bottom`**，`w:sz='18'`(2.25pt) |
| 字距 | `clean_text.replace(' ', '')` 仅清半角空格 | **`re.sub(r'\s+', '', clean_text)`** 清所有空白 |
| 图号清除 | `re.sub(r'[\(（]图\d+[\)）]', '', text)` | 不变 |
| 字体 | 楷体_GB2312, 18pt, 居中 | 不变 |

### 1.2 `w:sz` 厚度对照

| sz 值 | 点数 (pt) | 视觉效果 |
|:---:|:---:|------|
| 4 (T3) | 0.5pt | 极细线，与表格边框一致 |
| 12 | 1.5pt | 中等粗线 |
| **18 (T4)** | **2.25pt** | **显著粗线，视觉分隔感强烈** |

T4 选择 `sz='18'` (2.25pt)，使段落下边框具有明显的视觉分隔效果。这在 Word 中呈现为一条粗实的横线，类似 `<hr>` 但更可控。

### 1.3 `re.sub(r'\s+', '', clean_text)` 防御能力

T3 的 `clean_text.replace(' ', '')` 只能清除 ASCII 空格 (`U+0020`)。全角空格 (`U+3000`)、tab、换行符不会被清除。T4 的 `re.sub(r'\s+', '', clean_text)` 覆盖：`[ \t\n\r\f\v]`（ASCII 空白）+ 全角空格（Python `re` 的 `\s` 在 Python 3 中匹配 Unicode whitespace）。**防御性更强**。

### 1.4 修改清单

| 文件 | 函数 | 修改 |
|------|------|------|
| `patent_style_utils.py` | `add_section_title()` | 删除 `w:top` 循环，仅 `w:bottom` + `w:sz='18'` + 增强 whitespace 清除 |

## 2. Action Item 2: 末尾自动挂载说明书附图

### 2.1 现状分析

当前 v2.4 MD 中有 3 个 H1 标题：

```
# 说 明 书 摘 要
# 权 利 要 求 书
# 说 明 书
```

**缺失 `# 说明书附图`**。这是 AI 生成 MD 的常见瑕疵——模型未在末尾追加附图占位标题。T4 的自动挂载是一个务实的生产级兜底方案。

### 2.2 潜在重复风险

若未来 MD 中包含了 `# 说明书附图`，自动挂载会导致 DOCX 中出现**两份说明书附图标题**。建议增加去重检测：

```python
# 在令牌遍历主循环中，跟踪是否已检测到 # 说明书附图
has_drawing_title = False

if t.type == 'heading_open' and level == 1:
    if text == '说明书附图':
        has_drawing_title = True
    ...

# 主循环结束后
if not has_drawing_title:
    doc.add_page_break()
    add_section_title(doc, "说明书附图")
```

### 2.3 修改清单

| 文件 | 位置 | 修改 |
|------|------|------|
| `patent_md_to_docx.py` | `build_patent_docx()` 主循环 | 新增 `has_drawing_title` 检测标记 |
| `patent_md_to_docx.py` | `build_patent_docx()` return 前 | 新增兜底：分页 + `add_section_title(doc, "说明书附图")` |

## 4. 风险评估

| 风险项 | 严重度 | 说明 |
|--------|:---:|------|
| `w:sz='18'` 跨版本渲染 | P3 | 2.25pt 在不同 Word 版本/打印机下可能存在 ±0.25pt 的视觉偏差 |
| 附图标题重复 | P2 | 通过 `has_drawing_title` 标记防御，零风险 |
| 无附图内容时仅有标题 | P2 | 自动挂载只生成标题，无实际附图。需要后续人工插入图片。这是设计意图——物理兜底占位 |
| `\s+` 清除非汉字字符 | P3 | 若标题意外含英文/数字，也会被 `\s+` 清除其间的空格。但专利 H1 标题从未出现英文，风险可忽略 |

## 5. 验收标准对照

| T4 验收标准 | 达成方式 |
|------------|---------|
| H1 仅有底部边框，线条明显变粗 | 删除 `w:top`，`w:bottom` + `w:sz='18'`(2.25pt) |
| H1 文字绝对呈现 3 空格宽度，无粘连 | `re.sub(r'\s+', '', clean_text)` → `'   '.join()` 暴力重排 |
| 文档最后一页必有 `# 说 明 书 附 图` 大标题 | 主循环后兜底追加 `add_page_break()` + `add_section_title(doc, "说明书附图")` |
